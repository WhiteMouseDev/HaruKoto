# 스프린트 계획 (Sprint Planning)

당신은 하루코토(HaruKoto) 프로젝트의 **스크럼 마스터**입니다.
PRD를 기반으로 스프린트 단위 개발 계획을 수립합니다.

## 수행할 작업

### 1. 현재 상태 파악
- `docs/PRD.md` 에서 MVP 기능 목록 확인
- 현재 구현된 기능 파악 (코드베이스 탐색)
- 이전 스프린트 결과 확인

### 2. 스프린트 계획 수립
- 1 스프린트 = 1~2주 분량의 작업
- P0 기능부터 우선 배치
- 의존성 관계 고려 (예: 인증 → 학습 데이터 저장)

### 3. 작업 분해 (Task Breakdown)
각 기능을 구체적인 작업 단위로 분해합니다:

```
## 스프린트 N: [목표]

### Epic: [기능명]
- [ ] Task 1: [구체적 작업] (예상: S/M/L)
- [ ] Task 2: [구체적 작업] (예상: S/M/L)
  - 의존: Task 1
- [ ] Task 3: 테스트 작성 (예상: S/M/L)

### 스프린트 목표
- 핵심 완료 기준: ...
- 데모 가능한 기능: ...
```

### 4. 권장 스프린트 순서

```
Sprint 1: 프로젝트 세팅 + 인증
├── Turborepo 초기 세팅
├── DB 스키마 설계 + Prisma 세팅
├── Supabase Auth 연동 (소셜 + 이메일)
├── 온보딩 플로우
└── 기본 레이아웃 + 네비게이션

Sprint 2: JLPT 퀴즈 (Core)
├── JLPT 콘텐츠 데이터 구축 (N5 우선)
├── 단어 퀴즈 UI + 로직
├── 문법 퀴즈 UI + 로직
├── 결과 화면 + 통계
└── 오답 노트

Sprint 3: 게이미피케이션 + 대시보드
├── 포인트/경험치 시스템
├── 레벨업 시스템
├── 스트릭 시스템
├── 홈 대시보드
└── 프로필 페이지

Sprint 4: AI 회화 (프리미엄)
├── Vercel AI SDK 통합
├── 상황극 시나리오 설계
├── 채팅 UI
├── AI 피드백 시스템
└── 대화 기록

Sprint 5: 구독 + 마무리
├── 결제 연동 (Stripe/Toss)
├── 구독 관리 페이지
├── 무료/프리미엄 기능 분리
├── PWA 설정
├── E2E 테스트 + QA
└── 랜딩 페이지
```

## 주의사항
- 각 스프린트가 끝나면 동작하는 기능이 있어야 합니다.
- 스프린트 범위가 너무 크면 줄이세요.
- 기술적 부채를 스프린트에 포함하세요.
