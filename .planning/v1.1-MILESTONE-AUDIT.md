## Gap Details

### GAP-1: useTtsPlayer hardcoded Japanese toast (I18N-04, I18N-05)

**File:** `apps/admin/src/hooks/use-tts-player.ts` lines 78, 81
**Problem:** `toast.success('TTSを再生成しました')` and `toast.error(err.message || '再生成に失敗しました。もう一度お試しください。')` — hardcoded Japanese strings bypass i18n system.
**Locale keys exist but unused:** `tts.regenerateSuccess`, `tts.regenerateError` are defined in all 3 locale files.
**Fix:** Add `useTranslations('tts')` to hook, replace hardcoded strings with `t('regenerateSuccess')` / `t('regenerateError')`.

### GAP-2: hardcoded-strings.test.ts scope (test coverage)

**File:** `apps/admin/src/__tests__/hardcoded-strings.test.ts` line 21
**Problem:** Test only scans `.tsx` files (`entry.name.endsWith('.tsx')`). The `.ts` hook file escaped detection.
**Fix:** Extend to `.ts` files: `entry.name.endsWith('.tsx') || entry.name.endsWith('.ts')`.
