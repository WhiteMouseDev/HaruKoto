**A) FilterBar URL sync fix (P1-1):**

The search input uses `useState` initialized once from URL — does not update on popstate. The select dropdowns use `defaultValue` (uncontrolled). Fix both:

1. **Search input**: Keep `useState` for the debounce, but sync from URL on every render cycle. Add an effect that watches `searchParams.get('q')` and updates `searchValue` when the URL value differs (popstate/back/forward):

```tsx
// Replace the single useState init with:
const urlQ = searchParams.get('q') ?? '';
const [searchValue, setSearchValue] = useState(urlQ);

// Add sync effect for back/forward:
useEffect(() => {
  const urlQ = searchParams.get('q') ?? '';
  setSearchValue(urlQ);
}, [searchParams]);
```

But this creates a loop with the debounce effect. To fix, track whether the change is user-driven vs URL-driven. Simpler approach: use a ref to track "last pushed value" and only push URL when value differs from last push:

Actually, the cleanest fix: The debounce effect already compares `searchValue` and pushes URL. When URL changes via popstate, the sync effect sets `searchValue` to match URL, then debounce effect fires but since URL already matches, the `router.replace` is a no-op. This is fine.

However, the debounce effect has `[searchValue]` deps but also reads `searchParams` and `pathname`. To avoid the loop, keep the eslint-disable but add the URL sync effect:

```tsx
const urlSearch = searchParams.get('q') ?? '';
const [searchValue, setSearchValue] = useState(urlSearch);

// Sync search input with URL on back/forward navigation
useEffect(() => {
  const urlQ = searchParams.get('q') ?? '';
  if (urlQ !== searchValue) {
    setSearchValue(urlQ);
  }
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [searchParams]);
```

**A) Selection state reset on page/filter change (P1-3):**

In `content-table.tsx`, the `selectedIds` state persists across page changes because the component is never unmounted (same page, different URL params). Add a `useEffect` that clears selection whenever `data` reference changes (which happens on every new query result):

```tsx
import { useState, useEffect } from 'react';

// After the existing useState for selectedIds:
const [selectedIds, setSelectedIds] = useState<Set<string>>(new Set());

// Clear selection when data changes (page navigation, filter change)
useEffect(() => {
  setSelectedIds(new Set());
}, [data]);
```

This works because `data` is a new array reference on each query result. When page or filters change, TanStack Query returns new data, triggering the effect.

**B) Add placeholderData for smooth page transitions (P2):**

In `use-content-list.ts`, import `keepPreviousData` from TanStack Query and add it to the query options:

```tsx
import { useQuery, keepPreviousData } from '@tanstack/react-query';

// In the useQuery call, add placeholderData:
return useQuery<PaginatedResponse<T>>({
  queryKey: ['admin-content', type, params],
  queryFn: () => fetchAdminContent<T>(type, params),

```
