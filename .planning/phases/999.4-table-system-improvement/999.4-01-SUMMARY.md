# Phase 999.4 Plan 01: Fix Quiz API Contract + Real Pagination + Server-Side Sort Summary

Fix quiz API contract mismatch (P0-1), replace fake in-memory quiz pagination with SQL UNION ALL (P0-3), and add sort_by/sort_order to all four content list endpoints.

## What Was Built

### Task 1 — QuizAdminItem field rename

`QuizAdminItem.question_text` renamed to `sentence`. Since the field name is a single word, `CamelModel`'s alias generator produces `sentence` directly (no camelCase transformation needed). The frontend `QuizItem` type expects `sentence`, so this was a P0 contract mismatch causing blank quiz table cells.

### Task 2 — Real SQL quiz pagination + sort_by/sort_order

**Quiz list endpoint rewrite:** Replaced the previous approach (load ALL cloze + ALL sentence_arrange into Python, merge, slice) with a SQLAlchemy `union_all` of two projected `select` statements. Each projection exposes `(id, sentence, quiz_type, jlpt_level, review_status, created_at)`. A `COUNT(*)` on the combined subquery gives the accurate total, then `ORDER BY + OFFSET + LIMIT` does real DB-level pagination.

**sort_by/sort_order on all 4 endpoints:** Added `sort_by: str | None` and `sort_order: str` query params to `list_vocabulary`, `list_grammar`, `list_quiz`, `list_conversation`. Each endpoint has a `_XXXX_SORT_COLS` mapping. If `sort_by` is `None` or not in the mapping, defaults to `created_at`. `sort_order` defaults to `"desc"`.
