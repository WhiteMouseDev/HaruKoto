# Phase 999.4 Plan 02: FilterBar URL Sync + UX Fixes Summary

**One-liner:** Controlled select dropdowns and popstate sync effect for URL-driven FilterBar; showJlpt prop hides JLPT filter on conversation page; selection auto-clears on data change; keepPreviousData eliminates skeleton flash on pagination.

| 2 | Clear selection on data change + placeholderData keepPreviousData | da60f8d | content-table.tsx, use-content-list.ts |

## What Was Built

### Task 1: FilterBar URL sync + showJlpt prop

**P1-1 URL sync:** All three select dropdowns (JLPT, category, status) were changed from `defaultValue` (uncontrolled) to `value={searchParams.get(...) ?? ''}` (controlled). This makes them re-render with the correct value whenever URL changes via back/forward navigation. A `useEffect([searchParams])` was added to sync the search text input `searchValue` from URL when popstate fires.

**P1-2 JLPT hide:** Added `showJlpt?: boolean` prop to `FilterBarProps` (default `true`). The JLPT `<select>` block is now wrapped in `{showJlpt && (...)}`.

**P1-2 Conversation page:** `FilterBar` call in `conversation/page.tsx` now passes `showJlpt={false}`.

### Task 2: Selection reset + smooth pagination

**P1-3 Selection reset:** Added `useEffect(() => { setSelectedIds(new Set()); }, [data])` in `ContentTable`. Because TanStack Query returns a new array reference on every query result, this effect fires on every page or filter change, clearing stale checkbox selections.

**P2 Smooth pagination:** Imported `keepPreviousData` from `@tanstack/react-query` and added `placeholderData: keepPreviousData` to the `useQuery` call in `use-content-list.ts`. This shows the previous page's rows while the next page loads, preventing the skeleton flash.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `showJlpt` defaults to `true` | Preserves existing behavior for vocabulary/grammar/quiz pages; only conversation opts out |
| Controlled selects via `value=searchParams.get(...)` | Read from URL on every render — single source of truth, back/forward works automatically |
| `useEffect([data])` for selection reset | `data` is a new reference on each query result; no need to track page/filter params separately |
| `keepPreviousData` placeholder | Zero-config smooth pagination; previous page visible during fetch; no skeleton flash |
