## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default row padding (`p-3` = 12px rounded to this tier), card padding (`p-4`) |
| lg | 24px | Section vertical spacing |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (unused in this phase) |

Exceptions:
- Icon-only action buttons: fixed `size-8` (32px square) — established by existing TtsPlayer, matches touch-target minimum for desktop admin tool
- Waveform bar decorations inside skeleton: `w-1` width, height varies (12px/20px/16px)

## Typography

All sizes use `--font-sans` (system sans-serif) unless the text is Japanese content, which uses `.font-jp` (Noto Sans JP). This split is established in `globals.css`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Field name labels in TTS rows, helper text |
| Label-small | 12px (`text-xs`) | 400 (regular) | 1.5 | "No audio" placeholder text, muted helper copy |
| UI Label | 16px (`text-base`) | 600 (semibold) | 1.2 | Dialog titles, section headings |
| Display | 20px (`text-xl`) | 600 (semibold) | 1.2 | Page headings (not introduced in this phase) |

Weight constraint: exactly 2 weights used — 400 (regular) and 600 (semibold).

| Role | Token | Light Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | `#F9FAFB` | Page background, surfaces |
| Secondary (30%) | `--card` / `--secondary` | `#FFFFFF` / `#F3F4F6` | TTS player card container, row dividers |
| Accent (10%) | `--primary` | `#F6A5B3` (Cherry Pink) | Primary CTA "Generate" button (`variant="default"`), focus ring |
| Destructive | `--destructive` | `#EF4444` | Not used in this phase — regenerate is not destructive, it uses `variant="default"` |
| Status: has audio | `text-green-500` | Tailwind green-500 | `CheckCircle2` icon when field has audio |
| Status: no audio | `--muted-foreground` | `#6B7280` | `XCircle` icon when field has no audio |

Accent reserved for:
- "生成する" (Generate) button when a field has no audio (`variant="default"`)
- Focus ring on interactive elements (`--ring: #F6A5B3`)

The `--destructive` token is intentionally NOT used for regenerate — regeneration overwrites data but is reversible via re-regeneration, so it uses primary styling, not destructive.

## Interaction Contract

### TTS Row States

Each field row in `TtsPlayer` is independent after Phase 6. The row state machine:

```
[loading]  → skeleton pulse (shared for whole component while query is in-flight)
[no audio] → XCircle (muted) + "オーディオなし" (text-xs muted) + "生成する" (variant="default" sm)
[has audio, idle]    → CheckCircle2 (green-500) + field label + Play button (outline icon) + RotateCcw button (ghost icon)
[has audio, playing] → CheckCircle2 (green-500) + field label + Pause button (outline icon) + RotateCcw button (ghost icon)
[regenerating]       → Dialog open: Loader2 spin on confirm button, both dialog buttons disabled
```

### Per-Field Independence Rule

When `audios[fieldA]` is non-null and `audios[fieldB]` is null:
- Row A shows: CheckCircle2 + Play + RotateCcw
- Row B shows: XCircle + "オーディオなし" + "生成する"
- Playing row A does not affect row B's visual state
- Regenerating row A does not disable row B's buttons

### Loading Skeleton

The skeleton is shown for the entire `TtsPlayer` component while `ttsQuery.isLoading === true`. It matches the current skeleton layout: one pill placeholder + one circle placeholder + three waveform bars. This remains unchanged — skeleton is per-component, not per-field.

### Auto-play After Regeneration

After successful regeneration of a specific field, the hook auto-plays the new audio for that field only and sets `playingField` to that field's value. Other fields remain unaffected.

## Copywriting Contract

All copy is i18n-keyed via `next-intl`. No hardcoded strings in component files.

| Element | i18n Key | Japanese (ja.json) | Notes |
|---------|----------|-------------------|-------|
| Primary CTA (no audio) | `tts.generate` | "生成する" | `variant="default"` button |
| Status: no audio | `tts.noAudio` | "オーディオなし" | `text-xs text-muted-foreground` |
| Play button aria-label | `tts.play` | "再生" | Screen reader only |
| Pause button aria-label | `tts.pause` | "一時停止" | Screen reader only |
| Regenerate button aria-label | `tts.regenerateTooltip` | "再生成" | Screen reader only |
| Dialog title | `tts.regenerateTitle` | "{itemLabel}のTTSを再生成しますか？" | Interpolates itemLabel |
| Dialog confirm | `tts.regenerateConfirm` | "再生成する" | `variant="default"` |
| Dialog cancel | `tts.regenerateCancel` | "キャンセル" | `variant="outline"`, autoFocus |
| Dialog loading | `tts.regenerating` | "生成中..." | Shown inside confirm button when isPending |
| Success toast | `tts.regenerateSuccess` | "TTSを再生成しました" | via `sonner` toast.success |
| Error toast | `tts.regenerateError` | "再生成に失敗しました。もう一度お試しください。" | via `sonner` toast.error |
| Fetch error | `tts.fetchError` | "オーディオの読み込みに失敗しました" | via `sonner` toast.error on query error |

### New i18n Key Required (Phase 6 addition)

| Key | Japanese | Korean | English | Reason |
|-----|----------|--------|---------|--------|
| `tts.fields.exampleSentences` | "例文" | "예문" | "Example Sentences" | grammar `example_sentences` field label — new in Phase 6 (D-08) |

Note: `tts.fields.exampleSentence` (singular) already exists for vocabulary. The grammar field key is `exampleSentences` (plural) to match D-08 decision.

### Empty State

The TTS player itself is never shown in an "empty" state — it always renders for a content item. The per-field "no audio" state (`tts.noAudio`) IS the empty state for individual rows.

### Error State

| Scenario | Trigger | Copy | Behavior |
|----------|---------|------|----------|
| Query fetch fails | `ttsQuery.isError` | `tts.fetchError` → toast.error | Existing pattern — add error handling to hook |
| Regeneration fails | `regenerateMutation.onError` | `tts.regenerateError` → toast.error | Already implemented |

### Destructive Actions

No destructive actions in this phase. Regeneration overwrites a single field's audio but:
- Is reversible by regenerating again
- Uses `variant="default"` (primary), not `variant="destructive"`
- Confirmation dialog is retained to prevent accidental API calls
