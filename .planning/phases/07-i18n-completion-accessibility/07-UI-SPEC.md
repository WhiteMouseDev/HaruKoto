## Accessibility Contract (A11Y-01 to A11Y-04)

This section is the primary deliverable of Phase 7 on the interaction side.

### A11Y-01: aria-current="page" on Active Sidebar Nav Item

**File:** `apps/admin/src/components/layout/sidebar-nav-item.tsx`

**Current state:** `isActive` drives visual styling only (border-l-2, bg-accent). No ARIA attribute.

**Required change:** Add `aria-current={isActive ? 'page' : undefined}` to the `<Link>` element.

```
<Link
  href={href}
  aria-current={isActive ? 'page' : undefined}
  className={cn(...)}
>
```

No visual change. Attribute only.

### A11Y-02: Skip Link — "메인 콘텐츠로 건너뛰기"

**File:** `apps/admin/src/app/(admin)/layout.tsx`

**Required pattern:** Insert a visually hidden anchor as the FIRST child of the layout wrapper. Reveal on Tab focus using Tailwind's `focus:not-sr-only` pattern.

```
<a
  href="#main-content"
  className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:rounded-md focus:bg-background focus:px-4 focus:py-2 focus:text-sm focus:font-medium focus:ring-2 focus:ring-ring"
>
  {t('a11y.skipToMain')}
</a>
```

**Target:** `<main id="main-content" ...>` — add `id="main-content"` to the existing `<main>` element.

**i18n key:** `a11y.skipToMain` in all 3 locale files.

| Locale | Copy |
|--------|------|
| ko | 메인 콘텐츠로 건너뛰기 |
| ja | メインコンテンツへスキップ |
| en | Skip to main content |

### A11Y-03: Landmark aria-labels

**Files:** `apps/admin/src/app/(admin)/layout.tsx`, `apps/admin/src/components/layout/sidebar.tsx`

| Element | Current | Required |
|---------|---------|---------|
| `<aside>` in sidebar.tsx | No aria-label | `aria-label={t('a11y.sidebarNav')}` |
| `<nav>` in sidebar-nav-with-badges.tsx | No aria-label | `aria-label={t('a11y.primaryNav')}` |
| `<main>` in layout.tsx | No aria-label | `aria-label={t('a11y.mainContent')}` (optional per WCAG; include for clarity) |

All aria-label values must use i18n keys (Decision D-09 from CONTEXT.md).

| Key | ko | ja | en |
|-----|----|----|-----|
| `a11y.sidebarNav` | 사이드바 | サイドバー | Sidebar |
| `a11y.primaryNav` | 기본 탐색 | メインナビゲーション | Primary navigation |
| `a11y.mainContent` | 메인 콘텐츠 | メインコンテンツ | Main content |

### A11Y-04: Explicit label on Search Input

**File:** `apps/admin/src/components/content/filter-bar.tsx`

**Current state:** `<Input>` has `placeholder` only — no associated `<label>`.

**Required change:** Wrap with `<label>` or use `<label htmlFor="search-input">` + `id` on the input.

Preferred pattern (visually hidden label, preserves layout):

```
<label htmlFor="search-input" className="sr-only">
  {t('filter.searchLabel')}
</label>
<Input
  id="search-input"
  type="search"
  ...
/>
```

| Key | ko | ja | en |
|-----|----|----|-----|
| `filter.searchLabel` | 검색 | 検索 | Search |

## i18n Contract

### New Namespaces Required

| Namespace | Keys | Purpose |
|-----------|------|---------|
| `a11y` | skipToMain, sidebarNav, primaryNav, mainContent | Accessibility labels (Decision D-09) |
| `category` | travel, food, shopping, business, daily, etc. | Category label translation (Decision D-01) |
| `time` | justNow, minutesAgo, hoursAgo, daysAgo | Relative time in audit-timeline (Decision D-02) |
| `validation` | required, minLength, invalidFormat, etc. | Zod error messages (Decision D-03) |

### Hardcoded String Targets

Every hardcoded Japanese/Korean string in source .tsx files must be replaced with a `t()` call. The canonical target files are listed in CONTEXT.md under "Hardcoded Strings." For each target:

1. Identify the string and its semantic meaning.
2. Add a key to the appropriate namespace in ko.json, ja.json, en.json.
3. Replace the hardcoded string with `t('namespace.key')`.
4. Verify all 3 locale files have the key (D-11 validation script will enforce this).

### audit-timeline.tsx Time Expressions

**Current:** `formatRelativeTime()` returns hardcoded Japanese strings (`'たった今'`, `'${diffMin}分前'`, etc.).

**Required:** Replace with next-intl ICU MessageFormat using the `time` namespace.

| Key | ICU Pattern (example) |
|-----|----------------------|
| `time.justNow` | `{locale-specific "just now"}` |
| `time.minutesAgo` | `{count, plural, one {# minute ago} other {# minutes ago}}` (per locale) |
| `time.hoursAgo` | `{count, plural, one {# hour ago} other {# hours ago}}` |
| `time.daysAgo` | `{count, plural, one {# day ago} other {# days ago}}` |

Use `useTranslations('time')` client-side in AuditTimeline. Remove the `formatRelativeTime()` function entirely.

### Existing Namespace Additions

Append new keys to existing namespaces without restructuring them (Decision D-05):

| Namespace | New keys |
|-----------|----------|
| `filter` | `searchLabel` (for sr-only label on search input) |
| `a11y` | (new namespace — all keys new) |

### CI Validation Scripts (D-10, D-11)

Two scripts must be added (implementation approach at Claude's discretion per CONTEXT.md):

1. **Hardcoded string grep check:** Scans `apps/admin/src/**/*.tsx` for Japanese Unicode range `[\u3000-\u9FFF\uFF00-\uFFEF]` and Korean range `[\uAC00-\uD7AF]`. Exits non-zero if matches found outside the allowlist.
   - Allowlist: `locale-switcher.tsx` (displays locale names), any test/mock files.
2. **Key parity check:** Verifies ko.json, ja.json, en.json have identical key sets. Exits non-zero on divergence.
