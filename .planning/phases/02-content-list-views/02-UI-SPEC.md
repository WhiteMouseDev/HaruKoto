| Token | Value | Usage |
|-------|-------|-------|

## Color

Carried forward from Phase 1 globals.css. No new CSS variables introduced in Phase 2.
Status badge colors are defined in Phase 1 UI-SPEC and activated in this phase.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | #F9FAFB (--background) | #111827 | Page background, sidebar background |
| Secondary (30%) | #FFFFFF (--card) | #1F2937 | Cards, table surface, header bar, filter bar |
| Accent (10%) | #F6A5B3 (--primary) | #F6A5B3 | Active sidebar nav item indicator only |
| Destructive | #EF4444 (--destructive) | #F87171 | Error states only (no destructive actions in Phase 2) |
| Foreground | #111827 | #F9FAFB | All body and table text |
| Muted foreground | #6B7280 | #9CA3AF | Column headers, helper text, empty state body |
| Border | #E5E7EB | rgba(255,255,255,0.1) | Table borders, card borders, input borders |

Accent (#F6A5B3) is reserved for exactly these elements in Phase 2:
1. Active navigation item left-border indicator in sidebar (carried from Phase 1)
2. Sidebar active nav item background tint: use `--accent` (#FCE7EC light / #3D1F2A dark)
3. Dashboard progress bar fill (progress toward 100% approved)

Accent MUST NOT be applied to: table row hover, filter dropdowns, pagination buttons,
status badges, column sort indicators, or search input focus ring (use --ring, which is
already set to #F6A5B3 in globals.css — the ring variable is the focus ring, not accent fill).

### Status Badge Colors (Semantic — Not Accent)

These are semantic colors, not part of the 60/30/10 split. Use Tailwind utility classes
directly — do not add new CSS variables.

| Status | Background class | Text class | Hex values |
|--------|-----------------|------------|------------|
| needs_review | `bg-amber-100` | `text-amber-700` | #FEF3C7 / #B45309 |
| approved | `bg-green-100` | `text-green-700` | #D1FAE5 / #047857 |
| rejected | `bg-red-100` | `text-red-700` | #FEE2E2 / #B91C1C |

Dark mode: use `dark:bg-amber-900/30 dark:text-amber-400`,
`dark:bg-green-900/30 dark:text-green-400`, `dark:bg-red-900/30 dark:text-red-400`.

### Dashboard Progress Bar

- Track: `bg-border` (#E5E7EB)
- Fill: `bg-primary` (#F6A5B3) — this is the one permitted accent usage in Phase 2 beyond sidebar

## Interaction Contract

### Search (D-10)
- Input triggers search after 300ms debounce from last keystroke
- Search applies immediately (no submit button required)
- URL query param `q` updated via `useRouter().replace()` — no navigation event
- Empty search clears `q` param from URL

### Filters (D-11, D-12)
- Each dropdown selection applies immediately (no "Apply" button)
- URL query params updated: `jlpt`, `category`, `status`
- All four params (`q`, `jlpt`, `category`, `status`, `page`) are synced via URL
- Changing any filter resets `page` to 1

### Pagination (D-07)
- 20 items per page
- Page number buttons shown: first, last, current, 2 neighbors of current, ellipsis for gaps
- URL query param `page` (1-indexed)
- Active page button: `bg-primary text-primary-foreground` (cherry pink)

### Loading State
- Table rows replaced with skeleton rows (3 columns of `animate-pulse bg-muted rounded`)
- Skeleton rows: same 48px height as data rows
- Number of skeleton rows: 10 (half of page size — avoids layout shift)
- Filter bar remains interactive during loading

### Error State
- Table area replaced with centered error message + retry button
- Error message: Body (14px), muted-foreground
- Retry button: shadcn Button variant="outline"

### Empty State
- Table area replaced with centered icon + heading + body text
- Icon: `Search` (lucide) for filtered empty, `Database` for no-content empty, 48px, muted-foreground
- Heading: Heading (20px/semibold), foreground
- Body: Body (14px/regular), muted-foreground
