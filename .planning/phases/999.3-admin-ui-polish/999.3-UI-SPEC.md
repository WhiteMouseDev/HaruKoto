## Layout Contract

### Shell Structure

```
┌─────────────────────────────────────────────────────────┐
│  SIDEBAR (w-60, fixed left)     │  MAIN (flex-1)         │
│  ┌─────────────────────────┐   │  ┌────────────────────┐│
│  │ Logo + "HaruKoto Admin" │   │  │ Page content p-8   ││
│  │ h-14 border-b           │   │  │                    ││
│  ├─────────────────────────┤   │  └────────────────────┘│
│  │ Nav items (flex-1)      │   │                         │
│  │                         │   │  No top header bar.     │
│  │                         │   │  Sidebar owns identity. │
│  ├─────────────────────────┤   │                         │
│  │ LocaleSwitcher          │   │                         │
│  │ [displayName]           │   │                         │
│  │ LogoutButton            │   │                         │
│  └─────────────────────────┘   │                         │
└─────────────────────────────────────────────────────────┘
```

**Decision: Remove the top `<Header>` component entirely.**
- The sidebar already has `h-14` logo strip matching header height — visual alignment is preserved.
- User display name moves to sidebar bottom, above LogoutButton.
- Admin layout becomes: `flex min-h-screen` with sidebar + main only, no header row.

**Rationale:** The header bar currently duplicates identity and controls. At 1-3 user admin scale, a header is visual overhead. The sidebar contains all navigation and identity — it is the complete chrome.

## Typography

Font family: Noto Sans JP for all text. `font-jp` utility class applied at body level.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 | 1.5 | Body text, table cells, descriptions, badges |
| Label | 14px (text-sm) | 600 (font-semibold) | 1.4 | Nav item labels, card stat labels, button text |
| Heading | 20px (text-xl) | 600 (font-semibold) | 1.2 | Page titles (h1), card titles |

Source: `apps/admin/src/app/layout.tsx` — weight array is `['400', '600']` only.

## Badge Color Contract (fixes P3)

**Current problem:** `sidebar-badge.tsx` uses `bg-destructive` which renders as alarm-red `#EF4444`. A count of 12 pending reviews reads as an error state.

**New contract for `NavBadge`:**

```
bg: bg-primary/15  (cherry pink at 15% opacity — informational, not alarming)
text: text-primary  (#F6A5B3 — readable on light, same on dark)
border: none
size: h-5 min-w-5 rounded-full
font: text-[10px] font-semibold
```

In Tailwind classes: `ml-auto flex h-5 min-w-5 items-center justify-center rounded-full bg-primary/15 px-1 text-[10px] font-semibold text-primary`

For dark mode, `bg-primary/15` remains appropriate as primary is `#F6A5B3` (same in dark) — pink at 15% opacity on dark backgrounds reads as a soft informational tint.

**Comparison by semantic meaning:**

| Badge type | Color intent | Classes |
|------------|-------------|---------|
| needs_review count (sidebar) | Informational — items await action | `bg-primary/15 text-primary` |
| `needs_review` status (list rows) | Neutral status | existing `StatusBadge` component |
| `rejected` status (list rows) | Problem state | existing `StatusBadge` with destructive variant |
| Error notifications | Problem | `bg-destructive text-destructive-foreground` |

## Dashboard StatsCard Contract (fixes P4)

**Current problem:** Three stacked 28px numbers with identical styling. No visual differentiation between needs_review (urgent), approved (complete), and rejected (problematic). Reads like a raw data dump.

**New card layout:**

```
┌──────────────────────────────────────┐
│ [Icon]  Title                        │  ← CardHeader: icon + title
├──────────────────────────────────────┤
│  12     待確認  (needs_review)       │  ← amber/primary colored count
│  ──────────────────────────── ──     │
│  84     承認済  (approved)           │  ← green/success colored count
│  ──────────────────────────── ──     │
│  3      却下    (rejected)           │  ← muted colored count (not alarming)
│                                      │
│  ████████████████░░░░░░  87%        │  ← progress bar + label
└──────────────────────────────────────┘
```

**Count row color assignment:**

| Stat | Color class | Semantic |
|------|-------------|---------|
| needsReview number | `text-amber-500` | Action required — draws attention without alarm |
| approved number | `text-emerald-600` | Positive completion |
| rejected number | `text-muted-foreground` | Informational — rejected items are de-emphasized, not alarming |
| progress bar | `bg-primary` | Cherry pink brand fill |

**Card structure improvements:**
- Add a thin left border accent when needsReview > 0: `border-l-4 border-amber-400` — draws the reviewer's eye to cards needing attention
- Total count displayed as small label `/ {total}` next to approved count for context
- Count font remains 28px (`text-[28px]`) — size is appropriate, only color changes
- Stat label font: `text-sm text-muted-foreground` — unchanged

**Dashboard grid:** Keep `grid-cols-2 gap-6 max-w-[960px]` — layout is appropriate.

## Sidebar Bottom Zone Contract (fixes P1 + P2)

**Current problem:** Sidebar bottom renders `LocaleSwitcher` + `LogoutButton`. Header renders the same two controls plus user display name. Duplication.

**New sidebar bottom zone:**

```
┌──────────────────────────────────┐
│ border-t border-border           │
│ px-4 py-4 flex flex-col gap-2    │
│                                  │
│  [displayName]  ← new addition   │
│  text-xs text-muted-foreground   │
│                                  │
│  [LocaleSwitcher]                │
│  [LogoutButton]                  │
└──────────────────────────────────┘
```

`displayName` is the user's `full_name` from `user_metadata` or email prefix. It is already computed in both `header.tsx` and `dashboard/page.tsx` — move that logic into `Sidebar` via the `user` prop passed from `AdminLayout`.

**AdminLayout change:**
- Remove `<Header user={user} locale={locale} />` from the JSX
- Remove `Header` import
- The `user` prop is already available from `requireReviewer()` — pass it to `<Sidebar user={user} />`
- The outer `flex flex-col` wrapper becomes unnecessary; use `flex min-h-screen` directly with sidebar + main

**Sidebar prop addition:**
- `Sidebar` receives `user: User` prop
- Computes `displayName` internally
- Passes `user` to `SidebarBottom` sub-component (or inline in sidebar)
