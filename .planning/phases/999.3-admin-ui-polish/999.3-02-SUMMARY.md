# Phase 999.3 Plan 02: StatsCard Color-Coded Counts Summary

**One-liner:** Color-coded dashboard stats cards using amber/emerald/muted for needs_review/approved/rejected counts with conditional amber left border accent.

## What Was Built

Redesigned the `StatsCard` component to provide visual hierarchy among the three count types. Previously all three 28px numbers had identical styling (plain `font-semibold`), making the dashboard read as a raw data dump with no visual priority.

### Changes Made

**`apps/admin/src/components/features/dashboard/stats-card.tsx`**
- Added `cn` import from `@/lib/utils`
- `<Card>` now receives `className={cn(needsReview > 0 && 'border-l-4 border-l-amber-400')}` — thin amber left border draws reviewer attention to cards with pending items
- `needsReview` count span: added `text-amber-500` — action-required amber
- `approved` count span: added `text-emerald-600` — positive completion green
- `rejected` count span: added `text-muted-foreground` — de-emphasized gray
- Approved label now includes `<span className="ml-1 text-xs">/ {total}</span>` context label showing total items
