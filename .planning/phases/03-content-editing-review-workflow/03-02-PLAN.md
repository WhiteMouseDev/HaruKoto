    **A. GET single-item detail (4 endpoints):**

    ```python
    @router.get("/vocabulary/{item_id}", response_model=VocabularyDetailResponse)
    async def get_vocabulary_detail(item_id: uuid.UUID, db, reviewer) -> VocabularyDetailResponse:
        result = await db.execute(select(Vocabulary).where(Vocabulary.id == item_id))
        item = result.scalar_one_or_none()
        if item is None:
            raise HTTPException(status_code=404, detail="Not found")
        return VocabularyDetailResponse.model_validate(item)
    ```
    Same pattern for grammar (`Grammar`), cloze quiz (`ClozeQuestion`), sentence-arrange quiz (`SentenceArrangeQuestion`), conversation (`ConversationScenario`).

    For quiz, use two separate endpoints:
    - `GET /quiz/cloze/{item_id}` -> ClozeQuestionDetailResponse
    - `GET /quiz/sentence-arrange/{item_id}` -> SentenceArrangeDetailResponse

    **B. PATCH update (6 endpoints) per D-11:**

    For each content type, implement PATCH using `body.model_dump(exclude_unset=True, by_alias=False)`:
    - `PATCH /vocabulary/{item_id}` with VocabularyUpdateRequest
    - `PATCH /grammar/{item_id}` with GrammarUpdateRequest
    - `PATCH /quiz/cloze/{item_id}` with ClozeQuestionUpdateRequest
    - `PATCH /quiz/sentence-arrange/{item_id}` with SentenceArrangeUpdateRequest
    - `PATCH /conversation/{item_id}` with ConversationUpdateRequest

    CRITICAL: Use `by_alias=False` in `model_dump()` to get snake_case field names for `setattr()`.

    Each PATCH must:
    1. Fetch item or 404
    2. Compare old vs new values, build `changes` dict: `{field: {"before": old, "after": new}}`
    3. If changes exist, insert AuditLog(content_type=..., content_id=item_id, action="edit", changes=changes, reviewer_id=reviewer.id)
    4. Commit and refresh
    5. Return detail response

    Content type strings for AuditLog: "vocabulary", "grammar", "cloze", "sentence_arrange", "conversation"

    **C. POST review (5 endpoints) per D-13:**

    For each content type:
    - `POST /vocabulary/{item_id}/review` with ReviewRequest
    - `POST /grammar/{item_id}/review` with ReviewRequest
    - `POST /quiz/cloze/{item_id}/review` with ReviewRequest
    - `POST /quiz/sentence-arrange/{item_id}/review` with ReviewRequest
    - `POST /conversation/{item_id}/review` with ReviewRequest

    Each review endpoint must:
    1. Validate: if action == "reject" and reason is empty/None, raise HTTPException(422, "reason required for reject")
    2. Fetch item or 404
    3. Set item.review_status to ReviewStatus.APPROVED or ReviewStatus.REJECTED
    4. Insert AuditLog(content_type=..., content_id=item_id, action=body.action, reason=body.reason, reviewer_id=reviewer.id)
    5. Commit and refresh
    6. Return detail response

    **D. POST batch review (1 endpoint) per D-06:**

    `POST /batch-review` with BatchReviewRequest:
    1. Validate reject requires reason
    2. Map content_type string to SQLAlchemy model class
    3. For each id in ids: fetch item, update review_status, insert AuditLog
    4. Single commit for all
    5. Return OkResponse(count=len(ids))

    Content type mapping:
    ```python
    MODEL_MAP = {
        "vocabulary": Vocabulary,
        "grammar": Grammar,
        "cloze": ClozeQuestion,
        "sentence_arrange": SentenceArrangeQuestion,
        "conversation": ConversationScenario,
    }
    ```

    **E. GET audit logs (1 endpoint):**

    `GET /{type}/{item_id}/audit-logs` where type is vocabulary|grammar|quiz/cloze|quiz/sentence-arrange|conversation:
    - Query AuditLog where content_type matches and content_id == item_id
    - Order by created_at DESC
    - Return list[AuditLogItem]

    Use a single generic endpoint:
    ```python
    @router.get("/{content_type}/{item_id}/audit-logs", response_model=list[AuditLogItem])
    ```
    For quiz subtypes, accept "cloze" and "sentence_arrange" as content_type path params.

    **Import all new schemas at the top of the file.** Add uuid import if not present.
