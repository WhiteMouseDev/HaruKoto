## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot, inline badge padding |
| sm | 8px | Button internal padding (vertical), form field gap |
| md | 16px | Card body padding, form field spacing, section gap within a card |
| lg | 24px | Card-to-card gap, edit form section separation |
| xl | 32px | Page horizontal padding, toolbar-to-table gap |
| 2xl | 48px | Edit page top margin above action header |
| 3xl | 64px | Page-level vertical padding |

Exceptions:
- Checkbox touch target: 20px (Radix Checkbox default, acceptable at desktop-only scale)
- Audit log timeline left margin: 20px (aligns dot to label baseline)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form field labels, table cell text, audit log entry text |
| Label | 12px | 500 (medium) | 1.4 | Status badge text, section subheadings, muted metadata |
| Heading | 20px | 600 (semibold) | 1.2 | Edit page title (e.g., "単語を編集"), dialog title |
| Display | 24px | 700 (bold) | 1.2 | Not used in this phase — reserved for Phase 5 dashboard |

Enforced weights: 400 (regular) and 600 (semibold). 500 used only for badge/label chrome — not body prose.

Font note: Japanese content fields (word, reading, example_sentence, title_ja) render with `font-jp` class (`Noto Sans JP`). All UI chrome uses default system sans.

All values map to CSS custom properties defined in `apps/admin/src/app/globals.css`. Use semantic token names in code — never raw hex.

1. Primary "保存" (Save) button

Destructive reserved for:
1. "却下" (Reject) button — individual reject
2. Bulk reject CTA inside bulk action toolbar
3. "却下を確認" button inside the reject reason dialog

## Interaction States

### Save button ("保存する")
| State | Visual |
|-------|--------|
| Default | `variant="default"` (primary cherry pink) |
| Loading (mutation pending) | `disabled` + spinner icon (`Loader2` from lucide, `animate-spin`) |
| Success | Toast: 保存しました (green check) |
| Error | Toast: 保存に失敗しました。もう一度お試しください。 |

### Approve button ("承認")
| State | Visual |
|-------|--------|
| Default (not approved) | `variant="default"` (primary) |
| Already approved | `variant="outline"` + checkmark icon, muted label — not disabled but visually passive |
| Loading | `disabled` + Loader2 spinner |
| Success | StatusBadge updates to approved inline; Toast: 承認しました |

### Reject button ("却下")
| State | Visual |
|-------|--------|
| Default | `variant="destructive"` |
| Loading | `disabled` + Loader2 spinner |
| Opens | RejectReasonDialog opens on click |

### RejectReasonDialog
- Title: "却下の理由を入力してください"
- Textarea: placeholder "却下の理由を入力..." — required, minimum 1 character
- Confirm button: "却下する" — `variant="destructive"`, disabled until textarea non-empty
- Cancel button: "キャンセル" — `variant="outline"`
- On confirm: mutation fires; dialog closes; StatusBadge updates; Toast: 却下しました

### Bulk action toolbar
| State | Visual |
|-------|--------|
| No selection | Toolbar hidden (`hidden`) |
| 1+ selected | Toolbar visible: "選択 {N}件" + "承認" button (primary) + "却下" button (destructive) |
| Bulk reject | Fires same RejectReasonDialog — reason applied to all selected items |

### Form validation (submit-only per D-03)
| State | Visual |
|-------|--------|
| Valid submit | Mutation fires |
| Invalid field | `<FormMessage>` renders below the field in `text-destructive text-sm` |
| No inline real-time validation | — |

### Loading skeleton (edit page initial load)
- EditPageHeader area: `Skeleton` bars at heading size and button size
- Edit Form Card: `Skeleton` bars matching field row heights
- Audit Log: `Skeleton` with 3 timeline entry shapes
- Use shadcn `Skeleton` pattern (pulse animation): `animate-pulse bg-muted rounded`

## Accessibility Notes

- Checkbox: uses Radix CheckboxPrimitive — keyboard navigable, `aria-checked` state managed automatically.
- Dialog: Radix DialogPrimitive — focus trap, `aria-modal="true"`, `role="dialog"` applied automatically.
- Approve/Reject buttons: include `aria-label` with current status context when button text alone is ambiguous (e.g., `aria-label="承認: {item_title}"`).
- Form errors: `<FormMessage>` renders with `role="alert"` via shadcn's form component.
- Status badges in audit timeline: include `aria-label` matching action text for screen reader clarity.
