## Interaction Contract

### Field Row Layout (D-01)

Each content type renders a vertical stack of rows. One row per TTS field:
- vocabulary: 3 rows (иӘӯгҒҝж–№, еҚҳиӘһ, дҫӢж–Ү)
- grammar: 1 row (гғ‘гӮҝгғјгғі)
- cloze: 1 row (ж–Ү)
- sentence_arrange: 1 row (ж—Ҙжң¬иӘһж–Ү)
- conversation: 1 row (гӮ·гғҒгғҘгӮЁгғјгӮ·гғ§гғі)

Row internal layout (flex, items-center, gap-2):
```
[status icon 16px] [field name label flex-1] [optional: "гӮӘгғјгғҮгӮЈгӮӘгҒӘгҒ—" text] [action buttons]
```

Row with audio:
```
[CheckCircle2 text-green-500 size-4] [field name text-sm flex-1] [Play/Pause Button outline icon size-8] [RotateCcw Button ghost icon size-8]
```

Row without audio:
```
[XCircle text-muted-foreground size-4] [field name text-sm flex-1] [гҖҢгӮӘгғјгғҮгӮЈгӮӘгҒӘгҒ—гҖҚtext-xs text-muted-foreground] [з”ҹжҲҗгҒҷгӮӢ Button default sm]
```

Container wrapper: `rounded-lg border border-border bg-card p-3` (audio present) or `bg-muted` (audio absent for all rows). Row separator: `border-b border-border last:border-b-0`.

### Audio Playback (D-05)

- Each row has an independent Play/Pause button.
- `playingField: string | null` state tracks which row is currently playing.
- Clicking Play on a different row stops current playback and starts the new row (stop-then-play).
- Clicking Play on the currently playing row pauses it.
- Single `audioRef` instance shared across all rows (current API returns one audio URL per item).
- Waveform bars (`aria-hidden="true"`) animate with `animate-pulse` when `playingField !== null`.

### Regeneration Flow (D-04)

- Each row's RotateCcw or [з”ҹжҲҗгҒҷгӮӢ] button sets `confirmField: string | null` state.
- `RegenerateConfirmDialog` opens when `confirmField !== null`.
- On confirm: calls `regenerateTts(contentType, itemId, confirmField)`.
- On success: invalidates `['admin-tts', contentType, itemId]` query, auto-plays new audio, shows `toast.success`.
- On error: shows `toast.error`, closes dialog.
- `confirmField` resets to `null` on dialog close (cancel or after confirm).

### Loading State

Skeleton identical to existing pattern: animated pulse bars inside rounded-lg border container. No per-row skeleton вҖ” single container-level skeleton.

### State Machine Per Row

```
audio absent вҶ’ [з”ҹжҲҗгҒҷгӮӢ] вҶ’ confirmField set вҶ’ dialog open вҶ’ confirm вҶ’ loading вҶ’ audio present
audio present вҶ’ в–¶пёҸ вҶ’ playing (playingField = field) вҶ’ вҸёпёҸ вҶ’ stopped (playingField = null)
audio present вҶ’ RotateCcw вҶ’ confirmField set вҶ’ dialog open вҶ’ confirm вҶ’ loading вҶ’ audio refreshed
```

## API Constraint Note (for Executor)

The current backend `TtsAudio` model has a `(target_type, target_id, speed)` UniqueConstraint with no `field` column. This means one audio record per content item, not per field.

**Design resolution (agreed in RESEARCH.md):** The single audio URL is shared across all field rows. The audio-present vs audio-absent state is item-level, not field-level. All rows for an item either all show audio-present state or all show audio-absent state, based on whether `audioUrl` from `fetchTtsAudio` is non-null.

This is a known limitation. Per-field audio storage is a future backlog item outside Phase 999.1 scope.
