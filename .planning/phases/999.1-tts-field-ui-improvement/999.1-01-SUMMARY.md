**One-liner:** Replaced Select dropdown in TtsPlayer with vertical field list using CheckCircle2/XCircle status icons and per-row Play/Pause + Generate buttons, driven by playingField/confirmField string state in useTtsPlayer hook.

## What Was Built

- **useTtsPlayer hook** — replaced `selectedField`/`confirmOpen` boolean state with `playingField: string | null` and `confirmField: string | null`. `handlePlayPause(field: string)` stops any previously playing row before starting the clicked row. `regenerateMutation.mutationFn` accepts `field: string` and auto-plays after successful regeneration.

- **TtsPlayer component** — maps over `TTS_FIELDS[contentType].options` to render a vertical list of rows. Each row shows: status icon (CheckCircle2 green / XCircle muted), field name label, and action buttons (Play/Pause + RotateCcw when audio present; "No audio" text + Generate button when absent). Select dropdown completely removed.

- **Unit tests** — 9 test cases covering: field count per content type, audio-absent/audio-present state icons and buttons, click handlers for generate/play, dialog open state, loading skeleton, and absence of Select element.
