## TypeScript Compilation Issue

**Severity:** Warning (does not block runtime behavior or tests)

**File:** `apps/admin/src/__tests__/tts-player.test.tsx` line 53

**Issue:** The type cast `as ReturnType<typeof useTtsPlayer>['regenerateMutation']` for the mock `regenerateMutation` object causes a TS2352 error because the mock `{ mutate: Mock, isPending: false }` does not overlap sufficiently with the full `UseMutationResult<TtsAudioResponse, Error, string, unknown>` type.

**Impact:** Test file only. The component and hook compile without errors. Tests still execute and pass (9/9) because Vitest runs transpiled code, not strict TS. The mock itself only exercises `mutate` and `isPending`, both of which are provided.

**Classification:** Warning — not a blocker for phase goal, but should be resolved for strict compliance.

**Fix:** Replace the cast with `as unknown as ReturnType<typeof useTtsPlayer>['regenerateMutation']` per TS recommendation, or restructure the mock to satisfy the full `UseMutationResult` shape.
