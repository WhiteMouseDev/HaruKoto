### UX-03: Sidebar Badge Notification

**NavBadge appearance:**
- Shape: `h-5 min-w-5 rounded-full` ‚Äî pill/circle depending on digit count
- Background: `bg-destructive` (red ‚Äî signals pending work)
- Text: `text-[10px] font-semibold text-destructive-foreground`
- Inner padding: `px-1` (expands for 2+ digits)
- Max displayed: 99 ‚Äî values over 99 show "99+"
- Hidden: renders `null` when count is 0 (no empty badge)

**Badge placement in nav:**
- Inside `SidebarNavItem` link: appended after label text via `ml-auto`
- Layout: `flex items-center gap-2` with `<span>{label}</span>` then `<NavBadge count={badge} />`

**Data source:**
- Hook: `useDashboardStats()` (existing `use-dashboard-stats.ts`, `staleTime: 60_000`)
- Mapping: vocabulary‚Üívocabulary, grammar‚Üígrammar, quiz‚Üísum(cloze+sentence_arrange), conversation‚Üíconversation
- Refetch: page load + window focus (TanStack Query defaults ‚Äî no polling interval)

## Design Decisions from Claude's Discretion

(From CONTEXT.md ‚Äî discretion areas now resolved)

| Topic | Decision | Rationale |
|-------|----------|-----------|
| Review queue sort order | `created_at ASC` ‚Äî oldest first | Natural queue order; reviewers should address oldest pending items first |
| Prev/Next button position | Full-width strip between breadcrumb and ReviewHeader; Prev left, position center, Next right | Consistent with common content editor patterns; separates nav from review actions |
| Badge polling strategy | Page load + window focus only (`staleTime: 60_000`) ‚Äî no interval polling | Appropriate for 1-3 user scale; interval polling is overkill |
| Progress bar color | `bg-primary` (#F6A5B3 cherry pink) | Consistent with existing StatsCard implementation already shipping |
| Queue position display | "N / M" format (`text-sm text-muted-foreground`) centered in navigation bar | Compact, internationally readable without localization ambiguity |
