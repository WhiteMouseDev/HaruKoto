  <files></files>
  <action>
    Deploy apps/admin to Vercel using the Vercel CLI:

    1. **Check if Vercel CLI is available:**
       ```bash
       npx vercel --version
       ```
       If not available, install: `pnpm add -g vercel` or use `npx vercel`.

    2. **Link or create the Vercel project:**
       ```bash
       cd apps/admin && npx vercel link
       ```
       - If prompted for project settings:
         - Framework: Next.js
         - Root Directory: apps/admin (CRITICAL for monorepo)
         - Build Command: (default, let Vercel auto-detect)
         - Output Directory: (default)

    3. **Set environment variables via CLI:**
       ```bash
       npx vercel env add NEXT_PUBLIC_SUPABASE_URL production
       npx vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
       npx vercel env add SUPABASE_SERVICE_ROLE_KEY production
       ```
       Note: If the CLI prompts for values interactively, this will pause for user input. The user must provide the values from the Supabase Dashboard.

       Alternative if CLI env setting is not straightforward:
       - Set env vars via Vercel Dashboard -> Project -> Settings -> Environment Variables
       - This becomes a checkpoint:human-action step

    4. **Deploy:**
       ```bash
       cd apps/admin && npx vercel --prod
       ```

    5. **Verify deployment:**
       ```bash
       # Get the deployment URL
       npx vercel ls --limit 1
       ```
       - The URL should respond with a 200 (redirected to /login since not authenticated)
       - No MODULE_NOT_FOUND errors in build logs

    If the Vercel CLI requires authentication, this task will naturally pause for user login via `npx vercel login`.
