## Spacing Scale

Declared values (multiples of 4, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Button inner padding (vertical), list item gap |
| md | 16px | Card inner padding, form field gap, default element spacing |
| lg | 24px | Section padding, sidebar item group gap |
| xl | 32px | Page content horizontal padding |
| 2xl | 48px | Major section breaks, login card vertical padding |
| 3xl | 64px | Login page vertical centering offset |

Exceptions:
- Sidebar width: 240px (fixed, non-token value — desktop admin layout)
- Header height: 56px (fixed, aligns to 8pt grid)
- Locale switcher dropdown trigger: min-width 80px (language label container)

## Typography

Admin tool is desktop-only. Japanese is the primary rendering language — Noto Sans JP
must be loaded. Korean and English fall back to system sans-serif at same sizes.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Noto Sans JP |
| Label | 12px | 400 (regular) | 1.4 | Noto Sans JP |
| Heading | 20px | 600 (semibold) | 1.3 | Noto Sans JP |
| Display | 28px | 600 (semibold) | 1.2 | Noto Sans JP |

Weights in use: 400 (regular) and 600 (semibold) only. Do not introduce additional weights.
Form labels (Label row) are differentiated from body text by their smaller 12px size and
muted-foreground color (#6B7280), not by a separate weight. The login page app title at
Display 28px/semibold is visually dominant enough without requiring bold weight.

## Color

The admin app uses the same CSS variable architecture as apps/web but applies a
neutral-dominant palette (not the consumer cherry-pink spring theme). This signals
"tool" rather than "product" to the reviewer.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | #F9FAFB (gray-50) | #111827 (gray-900) | Page background, sidebar background |
| Secondary (30%) | #FFFFFF | #1F2937 (gray-800) | Cards, login form card, header bar |
| Accent (10%) | #F6A5B3 (cherry pink) | #F6A5B3 | Accent reserved for: login submit button, active sidebar nav item indicator, locale switcher active state |
| Destructive | #EF4444 | #F87171 | Access-denied error banner only |
| Foreground | #111827 (gray-900) | #F9FAFB (gray-50) | All body text |
| Muted foreground | #6B7280 (gray-500) | #9CA3AF (gray-400) | Helper text, secondary labels |
| Border | #E5E7EB (gray-200) | rgba(255,255,255,0.1) | Card borders, input borders, dividers |

Accent (#F6A5B3) is reserved for exactly three elements:
1. Login page "ログイン" submit button (primary action)
2. Active navigation item left-border indicator in sidebar
3. Locale switcher selected language pill/underline

Accent MUST NOT be applied to: hover states on list rows, neutral buttons, helper icons,
status badges, or any loading/skeleton state.

Status badge colors (separate semantic palette — not accent):
- needs_review: amber-100 background / amber-700 text (#FEF3C7 / #B45309)
- approved: green-100 background / green-700 text (#D1FAE5 / #047857)
- rejected: red-100 background / red-700 text (#FEE2E2 / #B91C1C)

Note: Status badges are defined here for reference; they are not rendered in Phase 1
but establish the contract all phases share.

## Layout Contract (Phase 1 Screens)

### Login Page — `/login`

- Full-viewport centered layout (flex items-center justify-center min-h-screen)
- Background: dominant (gray-50)
- Login card: 400px max-width, secondary (white), rounded-lg, border, shadow-sm
- Card padding: 48px (2xl token) vertical, 32px (xl token) horizontal
- App title at top of card: Display size (28px/semibold), centered
- Form fields stacked vertically with 16px (md token) gap
- Submit button: full-width, accent fill (#F6A5B3), Label size text, 40px height
- Error banner (access denied): destructive (#EF4444) background tint below the form,
  full card width, 8px padding, 12px text

### Admin Shell — `/(admin)/layout`

- Fixed left sidebar: 240px wide, dominant background (gray-50), border-right
- Sidebar sections: top = app logo + nav items; bottom = locale switcher + logout
- Header bar: full-width minus sidebar, 56px height, secondary background (white),
  border-bottom, flex justify-between items-center with 24px horizontal padding
- Main content area: flex-1, overflow-y-auto, 32px padding

### Dashboard Stub — `/(admin)/dashboard`

- Page heading: Heading size (20px/semibold)
- Empty state card: centered, max-width 480px, 48px vertical padding inside card,
  Display size emoji-free heading + Body size explanation + muted-foreground text
- Empty state card is the focal point of this screen; the heading text ("データがありません")
  is the primary visual anchor, rendered at Display size in foreground color
