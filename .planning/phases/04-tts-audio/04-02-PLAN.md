    **Step 2 — Add API functions to `apps/admin/src/lib/api/admin-content.ts`:**

    CRITICAL: FastAPI serializes Pydantic models with snake_case field names. The TypeScript type MUST use snake_case to match the actual JSON response. Using camelCase would result in `undefined` at runtime.

    Add type + two functions at the bottom of the file:
    ```typescript
    // IMPORTANT: Field names are snake_case to match FastAPI's Pydantic JSON serialization.
    // FastAPI AdminTtsResponse has `audio_url`, `field`, `provider` — NOT camelCase.
    export type TtsAudioResponse = {
      audio_url: string | null;
      field: string | null;
      provider: string | null;
    };

    export async function fetchTtsAudio(
      contentType: string,
      itemId: string
    ): Promise<TtsAudioResponse> {
      const supabase = createClient();
      const { data: { session } } = await supabase.auth.getSession();
      const res = await fetch(
        `${API_URL}/api/v1/admin/content/${contentType}/${itemId}/tts`,
        {
          headers: {
            Authorization: `Bearer ${session?.access_token ?? ''}`,
          },
        }
      );
      if (!res.ok) throw new Error('Failed to fetch TTS audio');
      return res.json();
    }

    export async function regenerateTts(
      contentType: string,
      itemId: string,
      field: string
    ): Promise<TtsAudioResponse> {
      const supabase = createClient();
      const { data: { session } } = await supabase.auth.getSession();
      const res = await fetch(`${API_URL}/api/v1/admin/tts/regenerate`, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          Authorization: `Bearer ${session?.access_token ?? ''}`,
        },
        body: JSON.stringify({
          content_type: contentType,
          item_id: itemId,
          field,
        }),
      });
      if (res.status === 429) {
        const data = await res.json();
        throw new Error(data.detail || 'Cooldown active');
      }
      if (!res.ok) throw new Error('TTS regeneration failed');
      return res.json();
    }
    ```

    **Step 3 — Create `apps/admin/src/hooks/use-tts-player.ts`:**

    Follow the hook skeleton from RESEARCH.md exactly. Key behaviors:
    - `useQuery` with key `['admin-tts', contentType, itemId]` calling `fetchTtsAudio`
    - `useState` for `isPlaying`, `selectedField` (defaulted from TTS_FIELDS), `confirmOpen`

    - `useMutation` calls `regenerateTts`. `onSuccess`: invalidate query, set localStorage cooldown, auto-play new audio via `audioRef.current = new Audio(data.audio_url); audioRef.current.play()`, fire success toast, close dialog, set remainingSeconds to 600
    - `onError`: fire error toast
    - `handlePlayPause()`: creates/reuses `Audio` object in `useRef`, toggles play/pause, sets `isPlaying`
    - `useEffect` cleanup: pause audio on unmount

    CRITICAL: All access to `TtsAudioResponse` fields must use snake_case: `data.audio_url` (NOT `data.audioUrl`). The hook reads `audio_url` from the query data and passes it through. The `audioUrl` returned by the hook to the component should be derived from `data?.audio_url`.

    The hook must have `'use client'` at top and return:
    ```typescript
    {
      audioUrl: data?.audio_url ?? null,  // Remap snake_case -> camelCase for component consumption
      isLoading, isPlaying, remainingSeconds, selectedField,
      setSelectedField, confirmOpen, setConfirmOpen, handlePlayPause,
      regenerateMutation, cooldownMinutes: Math.ceil(remainingSeconds / 60)
    }
    ```

    This way the snake_case boundary is contained in the data layer (API functions + hook internals), and the component interface uses idiomatic camelCase.
