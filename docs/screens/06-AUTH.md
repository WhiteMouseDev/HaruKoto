### 인터랙션 흐름

1. 소셜 로그인 → Supabase OAuth → 콜백 → (신규: 온보딩 / 기존: 홈)
2. 이메일 가입 → 인증 메일 발송 → 메일 확인 → 온보딩
3. 이메일 로그인 → 인증 성공 → 홈

## 데이터 의존성

| API/서비스 | 용도 |
|------------|------|
| Supabase Auth | OAuth (카카오, 구글) + 이메일 인증 |
| `POST /api/v1/user/profile` | 온보딩 데이터 저장 (닉네임, 레벨, 목표) |
| Zustand 스토어 | 온보딩 상태 관리 (step, 선택값) |
