# 작업 지시: PDF 단어 목록 → CSV 변환

> Claude App에서 실행할 작업. 169페이지 PDF를 분할 처리.
> 작업 완료 후 이 문서는 삭제할 것.

---

## 개요

일본어 단어 PDF (169페이지, 약 7000단어)를 Google Sheets용 CSV로 변환한다.

---

## PDF 분할 전략

169페이지를 한 번에 처리할 수 없으므로 **20~25페이지씩 나눠서** 처리한다.

| 배치 | 페이지 | 예상 단어 수 |
|------|--------|-------------|
| 1차 | 1~25 | ~1000 |
| 2차 | 26~50 | ~1000 |
| 3차 | 51~75 | ~1000 |
| 4차 | 76~100 | ~1000 |
| 5차 | 101~125 | ~1000 |
| 6차 | 126~150 | ~1000 |
| 7차 | 151~169 | ~800 |

---

## 각 배치에 사용할 프롬프트

PDF를 첨부하고, 페이지 범위를 지정하여 아래 프롬프트를 사용한다.

### 1차 배치 프롬프트 (1~25페이지)

```
첨부한 PDF의 1~25페이지에 있는 일본어 단어 목록을 CSV로 변환해주세요.

## 출력 형식
CSV 4개 컬럼 (헤더 포함):
reading,word,jlptLevel,meaningKo

## 변환 규칙

1. **reading**: 대괄호[] 앞의 히라가나
   - 예: あう[会う] → reading = あう
   - 한자가 없는 단어: あまり → reading = あまり

2. **word**: 대괄호[] 안의 한자 표기
   - 예: あう[会う] → word = 会う
   - 한자가 없는 단어: word = reading과 동일
   - 예: あまり → word = あまり
   - 카타카나 단어: word = 카타카나 그대로
   - 예: テレビ → word = テレビ, reading = テレビ

3. **jlptLevel**: N5, N4, N3, N2, N1 중 하나
   - PDF에 표시된 레벨을 그대로 사용

4. **meaningKo**: 한국어 뜻
   - 조사 힌트가 있으면 포함: (-に+) 만나다
   - 여러 뜻이 있으면 마침표/쉼표 구분 그대로 유지
   - 예: 딱 맞다. 서로 …하다

## 주의사항
- CSV 값에 쉼표가 포함되면 큰따옴표로 감싸주세요
- 빈 행 없이 연속으로 출력
- 헤더는 첫 번째 행에만
- 순서는 PDF에 나온 순서 그대로
- 누락 없이 모든 단어를 포함해주세요

## 예시 입출력

PDF 원본:
あう[会う] N5, (-に+) 만나다
あう[合う] N4, 딱 맞다. 서로 …하다
あかい[赤い] N5, 빨갛다
テレビ N5, TV

CSV 출력:
reading,word,jlptLevel,meaningKo
あう,会う,N5,"(-に+) 만나다"
あう,合う,N4,"딱 맞다. 서로 …하다"
あかい,赤い,N5,빨갛다
テレビ,テレビ,N5,TV
```

### 2차 이후 배치 프롬프트

```
이전에 이어서, 첨부한 PDF의 26~50페이지를 동일한 CSV 형식으로 변환해주세요.

형식: reading,word,jlptLevel,meaningKo
규칙은 이전과 동일합니다.

주의: 헤더(reading,word,jlptLevel,meaningKo)는 포함하지 마세요. 데이터만 출력해주세요.
```

> **중요:** 2차부터는 "헤더 제외"를 명시한다. 나중에 합칠 때 헤더가 중복되면 안 됨.

---

## 배치별 작업 순서

### Step 1: 배치별 실행

PDF를 분할할 필요 없다. **같은 PDF를 매번 첨부하고, 프롬프트에 페이지 범위만 바꾸면 된다.**

1. Claude App 새 대화 시작
2. **동일한 PDF 파일** 첨부
3. 1차 프롬프트 입력 (1~25페이지) → CSV 결과 복사
4. Google Sheets의 `vocabulary` 시트에 붙여넣기
5. 같은 대화에서 2차 프롬프트 입력 (26~50페이지) → 결과를 시트 마지막 행 아래에 붙여넣기
6. 3차~7차 반복 (페이지 범위만 변경)

### Step 2: 대화가 끊기는 경우

Claude 대화 컨텍스트가 길어지면 품질이 떨어질 수 있다.
**3~4 배치마다 새 대화를 시작**하는 것을 추천.

새 대화 시작 시:
1. 같은 PDF를 다시 첨부
2. 1차 프롬프트 형식(헤더 포함)을 사용하되, 페이지 범위만 변경

---

## Google Sheets 설정

### 시트 구조

**시트 이름: `vocabulary`**

| 열 | 헤더 | 입력 시점 | 설명 |
|----|------|----------|------|
| A | reading | PDF 변환 시 | 히라가나 읽기 |
| B | word | PDF 변환 시 | 한자 표기 (없으면 A와 동일) |
| C | jlptLevel | PDF 변환 시 | N5, N4, N3, N2, N1 |
| D | meaningKo | PDF 변환 시 | 한국어 뜻 |
| E | partOfSpeech | AI 보강 후 | NOUN, VERB 등 (처음엔 비워둠) |
| F | exampleSentence | AI 보강 후 | 일본어 예문 (처음엔 비워둠) |
| G | exampleReading | AI 보강 후 | 예문 읽기 (처음엔 비워둠) |
| H | exampleTranslation | AI 보강 후 | 예문 번역 (처음엔 비워둠) |

> E~H 열은 비워두고, A~D만 채운다.

### 붙여넣기 방법

1. Google Sheets에서 A1 셀 선택
2. 메뉴 → 파일 → 가져오기 → 업로드 → CSV 파일 업로드
   - 또는 A1에서 Ctrl+V로 직접 붙여넣기
3. 구분 기호: 쉼표(,)

CSV를 직접 붙여넣으면 한 셀에 전부 들어갈 수 있으므로,
**"데이터 → 텍스트를 열로 분할" 기능**을 사용하거나,
CSV 파일을 저장 후 "가져오기"로 올리는 게 확실하다.

## 검수 체크리스트 (모든 배치 완료 후)

### 1. 총 건수 확인
- 시트 하단에 `=COUNTA(A:A)-1` 수식으로 총 단어 수 확인
- 예상: 약 7000개 (±500)

### 2. 레벨별 건수 확인
- `=COUNTIF(C:C,"N5")` → N5 단어 수
- `=COUNTIF(C:C,"N4")` → N4 단어 수
- `=COUNTIF(C:C,"N3")` → N3 단어 수
- `=COUNTIF(C:C,"N2")` → N2 단어 수
- `=COUNTIF(C:C,"N1")` → N1 단어 수

### 3. 빈 셀 확인
- `=COUNTBLANK(A2:A8000)` → reading 빈 셀 수 (0이어야 함)
- `=COUNTBLANK(B2:B8000)` → word 빈 셀 수 (0이어야 함)
- `=COUNTBLANK(C2:C8000)` → jlptLevel 빈 셀 수 (0이어야 함)
- `=COUNTBLANK(D2:D8000)` → meaningKo 빈 셀 수 (0이어야 함)

### 4. 잘못된 레벨 확인
- `=COUNTIF(C:C,"<>N5")-COUNTIF(C:C,"N4")-COUNTIF(C:C,"N3")-COUNTIF(C:C,"N2")-COUNTIF(C:C,"N1")`
- 결과가 1이면 정상 (헤더), 2 이상이면 잘못된 레벨값 존재

### 5. 중복 확인
- 새 열(I열)에 `=A2&"|"&B2&"|"&C2` (reading+word+level 결합)
- `=COUNTIF(I:I,I2)>1` 로 중복 찾기
- 중복이 있으면 수동 확인 (진짜 중복인지, 다른 뜻인지)

### 6. Spot Check (샘플 대조)
- PDF에서 랜덤으로 10개 단어를 골라 시트와 대조
- 특히 페이지 경계(25, 50, 75... 페이지)에서 누락 확인

## 트러블슈팅

### Claude가 페이지를 못 읽는 경우
- 해당 페이지를 스크린샷으로 찍어서 이미지로 첨부
- 프롬프트: "이 이미지의 단어를 동일한 CSV 형식으로 변환해주세요"

### CSV에 쉼표가 깨지는 경우
- meaningKo에 쉼표가 포함된 경우 큰따옴표로 감싸져야 함
- Claude가 누락하면 수동으로 수정하거나 재요청

### 카타카나 단어의 reading
- 카타카나 단어(テレビ, コンピューター 등)는 reading도 카타카나 그대로
- 히라가나로 변환하지 않음 (てれび ✕ → テレビ ○)

### 한 단어에 여러 한자 표기가 있는 경우
- 예: おこなう[行う/行なう]
- 더 일반적인 표기 하나만 사용: word = 行う
- 두 번째 표기는 meaningKo에 메모: "행하다 (=行なう)"
