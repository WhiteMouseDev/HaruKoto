## 1. 프로젝트 개요

### 1.1 서비스 소개

**하루코토(ハルコト)**는 한국인을 위한 일본어 학습 앱입니다.

- **하루**: 한국어 "하루(하루)" + 일본어 "春(봄)"
- **코토**: 일본어 "言(말/단어)"
- **슬로건**: "매일 한 단어, 봄처럼 피어나는 나의 일본어"

JLPT(일본어능력시험) N5~N1 전 레벨 대비 + AI 실전 회화 연습을 제공합니다.

### 1.2 현재 상태

- **웹앱**: `app.harukoto.co.kr` (Next.js 기반, PWA 지원)
- **랜딩**: `harukoto.co.kr`
- **모바일 앱**: Flutter WebView 래퍼 (iOS/Android, 스토어 심사 준비 중)
- **개발 단계**: MVP 완료, 스토어 출시 직전

### 1.3 타겟 사용자

| 구분         | 설명                                                  |
| ------------ | ----------------------------------------------------- |
| **1차 타겟** | 일본어를 처음 시작하는 한국인 (10~30대)               |
| **2차 타겟** | JLPT 시험을 준비하는 학습자 (N5~N1)                   |
| **3차 타겟** | 일본 여행/비즈니스를 위해 회화를 연습하고 싶은 직장인 |

### 1.4 경쟁 서비스

| -------- | ---------------------------------- | -------------------- |
| Duolingo | 게이미피케이션, 다국어, 글로벌 1위 | 한→일 코스 품질 보통 |

## 3. 현재 브랜드 아이덴티티

### 3.1 서비스명 — "하루코토(ハルコト)"

**의도**: "하루(하루/봄)" + "코토(말)" = 매일 봄처럼 새로운 단어를 배운다

**현재 평가 (자체)**:

- 발음이 부드럽고 기억하기 쉬움
- 한국어와 일본어 양쪽 뉘앙스를 담음
- 다만 일본어를 모르는 사용자에게 "코토"가 직관적이지 않을 수 있음
- 검색 유니크성은 양호 (동일 서비스명 없음)

**디자이너님께 요청**: 네이밍 자체의 브랜드 파워, 발음 용이성, 확장성(글로벌 진출 시) 평가 부탁드립니다.

### 3.2 로고 시스템

현재 4가지 변형이 존재합니다:

| 변형          | 파일                  | 구성                           |
| ------------- | --------------------- | ------------------------------ |
| **심볼 마크** | `logo-symbol.svg`     | 벚꽃 꽃잎 5장 조합             |
| **가로형**    | `logo-horizontal.svg` | 심볼 + "하루코토" + "ハルコト" |
| **세로형**    | `logo-vertical.svg`   | 심볼 위, 텍스트 아래           |
| **워드마크**  | `logo-wordmark.svg`   | 텍스트만 (심볼 없이)           |

**로고 색상**: 단일 컬러 `#F59BA7` (체리 핑크), 그라데이션/스트로크 없음

**사용 폰트**:

- 한글 "하루코토": OTSBAggroM (산돌 아그로체)
- 일본어 "ハルコト": BM Euljiro OTF (배달의민족 을지로체)

**현재 평가 (자체)**:

- 심볼은 벚꽃 모티프가 서비스 컨셉과 잘 맞음
- 단일 컬러라 다양한 배경에 적용하기 어려움
- 워드마크의 두 폰트가 통일감 부족 (산돌 아그로 + 을지로체의 성격이 다름)
- 소형 사이즈(16px 이하)에서 심볼 가독성 미검증

### 3.3 마스코트 — 여우 "하루"

| 항목          | 설명                                                                                    |
| ------------- | --------------------------------------------------------------------------------------- |
| **캐릭터**    | 여우(きつね)                                                                            |
| **이름**      | 하루                                                                                    |
| **주요 색상** | `#FC6961` (코럴 레드 몸통), `#C3727E` (다크 로즈 디테일), `#E8DBDE` (배/얼굴 밝은 부분) |
| **사용 위치** | 로그인 화면, 가나 학습 완료 화면, 가나 매칭 완료 시 축하 등                             |
| **파일**      | `fox.svg`, `fox_charactor.svg` (2가지 변형)                                             |

**현재 평가 (자체)**:

- 캐릭터 자체는 귀엽지만, 브랜드와의 연결성이 약함 (왜 여우인지 스토리 부재)
- 사용 빈도가 낮아 브랜드 인지에 기여도가 낮음
- 벚꽃 핑크와 여우의 코럴 레드가 미묘하게 충돌

### 3.4 앱 아이콘

| 플랫폼           | 구성                                     | 배경색    |
| ---------------- | ---------------------------------------- | --------- |
| **PWA/브라우저** | 벚꽃 심볼 + "ハル" 텍스트, 라운드 사각형 | `#FFB7C5` |
| **iOS**          | PNG 아이콘 세트 (20~1024px)              | 확인 필요 |
| **Android**      | PNG mipmap (48~192px)                    | 확인 필요 |

**참고**: 앱 아이콘에는 "ハル" (하루의 일본어 표기)만 있고, 풀네임이나 한글은 없음

## 5. 타이포그래피

### 5.1 본문 폰트

```css
font-family: 'Noto Sans JP', 'Noto Sans KR', sans-serif;
```

- **일본어**: Noto Sans JP (Google Fonts)
- **한국어**: Noto Sans KR (Google Fonts)
- **선정 이유**: 한·일 혼합 텍스트에서 글리프 호환성이 좋음

### 5.2 로고 폰트 (앞서 언급)

- 한글: OTSBAggroM (산돌 아그로체)
- 일본어: BM Euljiro OTF (배달의민족 을지로체)

### 5.3 현재 평가 (자체)

- 본문 Noto Sans는 안전한 선택이지만, 학습 앱에 특별한 개성을 부여하지 못함
- 타이포그래피 위계(heading, body, caption)가 시스템화되어 있지 않음
- 일본어 학습 앱이므로 일본어 문자(히라가나/가타카나/한자)의 가독성이 특히 중요

## 6. UI 컴포넌트 현황

### 6.1 레이아웃

```
┌──────────────────────┐
│     Status Bar       │
├──────────────────────┤
│                      │
│                      │
│   Main Content       │
│   (max-width: 512px) │
│   (centered)         │
│                      │
│                      │
├──────────────────────┤
│   Bottom Navigation  │
│   ┌──┬──┬──┬──┬──┐   │
│   │홈│학습│채팅│복습│MY│  │
│   └──┴──┴──┴──┴──┘   │
└──────────────────────┘
```

- 모바일 퍼스트 (max-width 512px 중앙 정렬)
- 하단 고정 네비게이션 5탭 (Framer Motion 애니메이션)
- 컨텐츠 영역 `pb-24` 패딩으로 네비와 겹침 방지

### 6.2 주요 컴포넌트

| 컴포넌트         | 스타일                         | border-radius          |
| ---------------- | ------------------------------ | ---------------------- |
| 카드             | 흰색 배경, 핑크 보더 `#FCE7EC` | `rounded-2xl` (1rem)   |
| 버튼 (Primary)   | `#F6A5B3` 배경, 흰색 텍스트    | `rounded-xl` (0.75rem) |
| 버튼 (Secondary) | `#FFF0F3` 배경                 | `rounded-xl`           |
| 입력 필드        | 핑크 보더 `#FCE7EC`            | `rounded-xl`           |
| 뱃지/태그        | `#FFF0F3` 배경                 | `rounded-full`         |
| 프로그레스 바    | `#F6A5B3` fill                 | `rounded-full`         |
| 탭               | 커스텀 탭 (Radix 미사용)       | `rounded-full`         |

### 6.3 아이콘 시스템

- **Lucide React** 아이콘 라이브러리 사용
- 선형(outline) 스타일 통일
- 커스텀 아이콘 없음

### 6.4 애니메이션

- **Framer Motion** 사용
- 페이지 전환: fade + slide
- 카드: hover scale, tap scale
- 하단 네비: spring 물리 기반 인디케이터
- 퀴즈: 정답/오답 피드백 애니메이션

- 슬로건: "매일 한 단어, 봄처럼 피어나는 나의 일본어"
- 카카오 로그인 버튼 (노란색 `#FEE500`)
- 이메일 로그인/회원가입 버튼
- 배경: 위→아래 그라데이션 (`#FCF6F5` → `#FFF0F3`)

### 7.2 온보딩 (3~4단계)

1. **레벨 선택**: JLPT N5~N1 카드 리스트
2. **가나 학습 여부** (N5 선택 시): 히라가나/가타카나 학습 권유

- 추천 학습 카드 (단어/문법/청해)

### 7.4 학습 탭

- 학습 유형별 카드: 단어, 문법, 청해
- 각 카드에 진도 표시
- 가나 학습 배너 (상단)

### 7.5 채팅 탭 (AI 회화)

## 8. 기술적 제약사항

디자인 작업 시 아래 기술적 제약을 고려해 주세요.

### 8.1 프레임워크 제약

| 항목          | 현재 사용                    | 비고                                 |
| ------------- | ---------------------------- | ------------------------------------ |
| UI 라이브러리 | shadcn/ui (Radix Primitives) | 커스터마이징 자유도 높음             |
| CSS           | Tailwind CSS v4              | 유틸리티 기반, 디자인 토큰 변경 용이 |
| 아이콘        | Lucide React                 | 다른 아이콘 세트로 교체 가능         |
| 애니메이션    | Framer Motion                | 복잡한 애니메이션 구현 가능          |
| 폰트          | Google Fonts                 | 다른 웹폰트로 교체 가능              |

### 8.2 모바일 앱 제약

- **WebView 래퍼**: 네이티브 UI가 아닌 웹 UI를 그대로 표시
- **Safe Area**: iOS 노치/다이나믹 아일랜드, 홈 인디케이터 영역 고려 필요
- **Apple 심사 리스크**: 단순 WebView 래퍼는 Apple 가이드라인 4.2 위반 가능성 — 네이티브 느낌의 UI가 중요

### 8.3 반응형

- 기본 모바일(~512px) 최적화
- 태블릿/데스크톱은 중앙 정렬 + 최대 너비 제한
- PWA로도 동작하므로 홈 화면 아이콘 품질 중요

### 8.4 다크 모드

- 라이트/다크 모드 모두 지원 필수
- CSS 커스텀 프로퍼티 기반으로 테마 전환
- 라이트 모드가 기본 (봄 테마)

## 9. 개선이 필요하다고 느끼는 부분 (자체 진단)

> 아래는 개발팀의 주관적인 의견입니다. 전문가 관점에서 우선순위를 재조정해 주세요.

### 9.1 브랜딩 레벨

1. **색상 일관성 부재**: 핑크 계열 값이 3~4개 혼용 (`#F59BA7`, `#F6A5B3`, `#FFB7C5`, `#FFF0F3`)
2. **로고 폰트 부조화**: 한글(아그로체)과 일본어(을지로체)의 캐릭터가 다름
3. **마스코트 활용도 낮음**: 여우 캐릭터가 앱 내에서 거의 보이지 않음
4. **브랜드 스토리 부재**: "왜 벚꽃인지", "왜 여우인지" 설명할 수 있는 내러티브 없음

### 9.2 UI/UX 레벨

1. **전문적이지 않은 느낌**: 전반적으로 "개발자가 만든 UI" 느낌
2. **시각적 위계 부족**: 중요한 정보와 부수적 정보의 시각적 차이가 불분명
3. **여백/간격 시스템 미정립**: 일관된 스페이싱 규칙이 없음
4. **일러스트/그래픽 부재**: 텍스트 위주로 시각적 흥미 요소 부족
5. **온보딩 경험**: 첫 사용자에게 충분한 가치 전달이 되는지 불확실
6. **게이미피케이션 시각 디자인**: 배지, 스트릭, 레벨 등의 보상 요소가 시각적으로 매력적이지 않음

### 9.3 앱 스토어 레벨

1. **앱 아이콘**: 현재 아이콘이 스토어에서 눈에 띄는지 불확실
2. **스크린샷 디자인**: 스토어용 스크린샷 프레이밍이 필요
3. **피처 그래픽**: Google Play용 1024x500 그래픽 미제작
