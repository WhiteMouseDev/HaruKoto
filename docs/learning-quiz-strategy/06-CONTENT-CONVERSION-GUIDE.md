# 콘텐츠 변환 가이드

> 최종 수정: 2026-03-20
> 참조 사이트: Bunpro (문법 중심 일본어 학습)
> 목적: Bunpro의 문법 커리큘럼을 우리의 "단어+문법 통합 마이크로 레슨"으로 변환

---

## 1. 변환 원칙

```
1. 구조(커리큘럼 순서, 문법 관계)는 참조한다
2. 문장(예문, 설명)은 신규 작성한다 (저작권)
3. 문법 중심 → 단어+문법+대화문 통합으로 변환한다
4. Bunpro 1레슨(12~13문법) → 우리 7~10레슨으로 분해한다
```

---

## 2. 입력 (PM이 제공하는 것)

### 필수
- Bunpro N5 전체 레슨 목록 스크린샷 (Lesson 1~10, 문법 포인트 목록)
- 이것만 있으면 매핑 테이블 생성 가능

### 선택 (필요 시)
- 개별 문법 포인트 상세 페이지 스크린샷
- 대화문/문제 생성 시 참고용
- 특히 Caution, Synonyms/Related 정보가 유용

### 파일명 규칙
- 자유 (내용으로 판단)
- 순서만 맞으면 됨

---

## 3. 스크린샷에서 추출하는 것

| 추출 항목 | 예시 | 용도 |
|----------|------|------|
| 문법 포인트 | です, は, も | 우리 문법 DB 매핑 |
| 의미/기능 | "To be, Is" | 한국어 뜻 참조 |
| 품사 | Auxiliary Verb, Particle | 분류 |
| 구조 | Noun + です | 활용 패턴 |
| 예문 패턴 | XはYです, XもYです | 대화문 템플릿 참조 (문장 복제 X) |
| Synonyms | です ↔ だ ↔ である | 지식 그래프 연결 |
| Antonyms | です ↔ じゃない | 지식 그래프 연결 |
| Related | です → んです | 선행/후속 관계 |
| Caution | い형용사 뒤에 です 직접 불가 | 오답 선택지/피드백 설계 |
| 레슨 순서 | Lesson 1: 1/12, 2/12... | 커리큘럼 순서 참조 |

## 4. 매핑 규칙

### Bunpro → 우리 레슨 변환

```
Bunpro 1레슨 (12~13개 문법)
  ↓
의미적으로 관련된 문법끼리 그룹핑
  ↓
1그룹 = 우리 1레슨 (문법 1개 + 단어 5~8개)
  ↓
결과: Bunpro 1레슨 → 우리 7~10레슨
```

### 그룹핑 기준

| 기준 | 예시 |
|------|------|
| 정중체/보통체 쌍 | です + だ |
| 긍정/부정 쌍 | ます + ません |
| 지시사 묶음 | これ + それ + あれ |
| 조사 묶음 | は + も (단, 1레슨에 조사 1개) |
| 형용사 유형 | い형용사, な형용사 (각각 1레슨) |

### 문법-단어 엮는 기준

| 문법 | 어울리는 단어 유형 | 예시 |
|------|-----------------|------|
| です | 정체성/분류 명사 (직업, 국적) | 学生, 先生, 日本人 |
| は | 토픽이 쉬운 명사 (사람/사물) | 私, あなた, 田中さん |
| も | 쌍으로 바꿔치기 가능한 단어 | 私/あなた, 学生/先生 |
| これ/それ/あれ | 가리킬 수 있는 사물 | ペン, 本, かばん |
| の | 소유/관계 표현 | 私の, 田中さんの |
| い형용사 | い형용사 5개 | おいしい, たかい, やすい |
| な형용사 | な형용사 5개 | しずか, げんき, べんり |

### 단어 선정 기준 (우선순위)

```
1. 기존 N5 606개에 있는 단어 우선 (order 번호 매핑)
2. 해당 문법과 자연스럽게 쓰이는 단어
3. 이미지화 쉬운 구체적 단어 (추상어보다 구상어)
4. 다음 2~3레슨에서 재사용 가능한 단어
5. 형태가 단순한 단어 (초반일수록)
```


## 6. 대화문 생성 규칙

### Bunpro에는 대화문이 없으므로 신규 작성

```
규칙:
  1. 3~4턴 대화
  2. 타깃 문법이 최소 2회 등장
  3. 타깃 단어 5개 중 최소 4개 등장
  4. 화자 2명, 각각 voice_id 배정
  5. 자연스러운 상황 설정 (일상/직장/학교/여행)
  6. N5 범위 밖 단어 사용 금지 (예외: 고유명사)
```

### 대화 상황 선정 기준

```
문법에 따라 자연스러운 상황이 다름:
  です → 자기소개, 물건 설명
  は → 주제 제시, 비교
  も → 동의, 공통점 찾기
  か → 질문, 확인
  これ/それ/あれ → 쇼핑, 교실
  てください → 요청, 안내
```

## 7. 자동 QA 규칙

| 체크 항목 | 기준 | 실패 시 |
|----------|------|--------|
| 단어 수 | 5~8개 | 조정 |
| 문법 수 | 1개 | 조정 |
| 대화 턴 수 | 3~4턴 | 조정 |
| 문제 수 | 5개 (인식2+적용2+산출유사1) | 재생성 |
| N5 범위 | 범위 밖 단어 0개 | 교체 |
| 오답 충돌 | 정답과 뜻 겹침 0개 | 교체 |
| 문법 등장 | 대화문에 2회 이상 | 재생성 |
| 단어 커버리지 | 5개 중 4개 이상 등장 | 조정 |

## 8. PM 검수 체크리스트

```
□ 대화문이 자연스러운 일본어인가?
□ 한국어 번역이 정확한가?
□ 문법 설명이 초보자에게 이해 가능한가?
□ 단어 선정이 이 레슨 주제에 적합한가?
□ 문제 난이도가 적절한가? (너무 쉽거나 어렵지 않은지)
□ 전체 흐름이 10분 안에 완료 가능한가?
```

---

## 9. 배치 운영

```
1순위: Bunpro N5 전체 매핑 테이블 (1회)
  → PM: 레슨 목록 스크린샷 제공
  → AI: 매핑 테이블 생성
  → PM: 검수 (1~2시간)

2순위: Chapter 단위 배치 생성 (5레슨씩)
  → AI: 블루프린트 + 레슨 콘텐츠 자동 생성
  → Codex: 검증
  → PM: 최종 검수

3순위: 전체 N5 완성 후 N4 확장
```

---

## 10. Bunpro 데이터 활용 범위

```
✅ 활용하는 것:
  - N5 문법 목록과 순서
  - 문법 간 관계 (Synonyms/Antonyms/Related)
  - 문법 구조 (Structure: Noun + です)
  - Caution 사항 (오답/피드백 설계에 활용)

❌ 활용하지 않는 것:
  - 예문 텍스트 (신규 작성)
  - 설명 텍스트 (우리 스타일로 재작성)
  - UI/UX 구조 (우리는 마이크로 레슨)
```
