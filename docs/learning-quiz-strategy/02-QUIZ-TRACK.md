# 퀴즈 트랙 (Quiz Track) 설계

> 최종 수정: 2026-03-20
> 상태: 설계 확정 (일부 미결 사항 하단 참조)

---

## 1. 목적

**"출퇴근 길에 5분만 퀴즈를 풀면서 일본어와 친숙해지는 경로"**

- 대상: 유저 B "가벼운 퀴즈 유저" — 커리큘럼 없이 퀴즈만 풀면서 배우는 사람
- 목표: 빠르게 풀고 성취감, 틀리면서 배움, 단어/문법과 친숙해지기
- 학습 탭을 안 해도 **퀴즈 탭만으로 독립 학습 가능**
- 학습 탭을 한 유저는 **SRS 복습**이 퀴즈 탭에서 진행됨

---

## 2. 퀴즈 탭 카테고리: 4탭

```
[단어]  [문법]  [한자]  [리스닝]
```

각 탭 진입 → "오늘의 퀴즈" 원버튼 → 엔진이 구성 → 바로 시작

---

## 3. 탭별 퀴즈 스타일

### [단어]

| 모드 | 비율 | 상호작용 | 설명 |
|------|------|---------|------|
| 4지선다 | 80% | 탭 (한손) | JP→KR, KR→JP 섞어서 |
| 문맥 빈칸 선택 | 20% | 탭 (한손) | 예문 속 빈칸에 맞는 단어 선택 |

- 숙련도에 따라 방향 자동 조절:
  - 처음 본 단어: JP→KR (쉬운 방향)
  - 2~3회 정답: KR→JP 섞기 시작
  - 5회 이상 정답: KR→JP 위주
- 비율은 적응형으로 운영:
  - 입문: 80/20
  - 중급: 70/30
  - 고급: 60/40

### [문법]

| 모드 | 비율 | 상호작용 | 설명 |
|------|------|---------|------|
| 문장 재배열 | 60% | 드래그 | 타일을 올바른 순서로 배열 |
| 조사 선택 | 40% | 탭 | 빈칸에 は/が/を/に 등 선택 |

- 양손 편할 때 추천

### [한자]

| 모드 | 비율 | 상호작용 | 설명 |
|------|------|---------|------|
| 4지선다 (읽기) | 100% | 탭 (한손) | 한자 보고 올바른 읽기 선택 |

- 한국인 특화: 한자음 브릿지(식↔しょく) 활용 선택지
- N5에서는 가벼운 수준, N4~부터 본격 확대

### [리스닝]

| 모드 | 비율 | 상호작용 | 설명 |
|------|------|---------|------|
| 지문/대화 듣고 답 선택 | 100% | 탭 | 문장·대화 형태 음성 후 4지선다 |

- 이어폰 필요 — 사용자가 직접 탭 선택해서 진입
- 기본 퀴즈 세션에는 포함되지 않음

#### 리스닝 콘텐츠 소스

```
Phase 1: 레슨 대화문 재활용
  → lessons.content_jsonb의 대화문을 리스닝 퀴즈로 출제
  → 별도 콘텐츠 제작 불필요
  → 학습 탭에서 배운 대화를 리스닝으로 복습하는 경험

Phase 2 (추후): 리스닝 전용 콘텐츠 추가
  → 레슨과 독립적인 리스닝 지문
  → 뉴스, 안내 방송 등 다양한 형태
```

#### 대화문 음성 (TTS)

- 화자별 다른 목소리 필수 (ElevenLabs)
- 대화 인원 수만큼 voice_id 설정
- 최소 2개 음성: 남성(japanese_male_1) + 여성(japanese_female_1)

```json
"script": [
  {"speaker": "A", "voice_id": "japanese_male_1", "text": "すみません、これはいくらですか。"},
  {"speaker": "B", "voice_id": "japanese_female_1", "text": "300円です。"}
]
```

#### 리스닝 난이도

| 레벨 | 형태 | 예시 |
|------|------|------|
| 쉬움 | 짧은 문장 1개 | 🔊 "りんごをください" → 무엇을 달라고 했나요? |
| 보통 | 대화 2~3턴 | 🔊 A: "どこに行きますか" B: "えきに行きます" → B는 어디? |
| 어려움 | 대화 3~4턴 + 추론 | 🔊 긴 대화 → 다음에 무엇을 할까요? |

## 4. 퀴즈 UX 원칙

### "오늘의 퀴즈" 원버튼

- 유저는 탭 선택 → [시작]만 누름
- 모드 선택 없음 — 엔진이 비율대로 구성
- 문항 수: 기본 20문항 (약 5분)

### 정답/오답 후 피드백 화면

```
┌─────────────────────────────┐
│  ✓ 正解!  たべる = 먹다      │
│                              │
│  📖 食べる (たべる)           │
│  [동사] 먹다                  │
│                              │
│  例) パンを食べます。          │
│      빵을 먹습니다.           │
│                              │
│  ─────────────────────────  │
│  🔊 다시듣기   📌 단어장     │
│  📝 오답노트   ▶ 다음 문제   │
└─────────────────────────────┘
```

- 맞아도 틀려도 **항상** 단어 정보 + 예문 표시
- 찍어서 맞춘 경우에도 학습 기회 제공
- 단어장/오답노트 추가 가능

### 맞춤 연습 (파워유저용)

- 하단에 조용히 배치
- 퀴즈 모드 선택 (매칭 등 추가 모드 포함)
- 범위 선택 (레벨 / 챕터)
- 문항 수 선택

---

## 5. 오답 선택지(Distractor) 설계

### 원칙: 랜덤 오답 금지

| 난이도 | 오답 유사도 | 예시 (정답: たべる/먹다) |
|--------|-----------|----------------------|
| Level 1 (쉬움) | 의미 완전히 다름 | 크다 / 빠르다 / 예쁘다 |
| Level 2 (보통) | 같은 카테고리 | 마시다 / 만들다 / 사다 |
| Level 3 (어려움) | 형태 유사 | たてる / たびする / たずねる |
| Level 4 (최상) | 의미+형태 모두 유사 | きく(聞)/きく(効)/きる(切)/きる(着) |

### 오답 생성에 필요한 데이터

- 어휘 메타: 품사, 활용형, 읽기, JLPT 레벨, 빈도
- 의미 유사도: 같은 의미장(semantic field) 태깅
- 형태 유사도: 표기(한자/가나) 편집거리
- 혼동 로그: 사용자 오답 이력 기반 (운영 후 축적)

---

## 6. 처음 보는 단어 처리

- 퀴즈 탭에서 처음 만나는 단어 = 틀려도 OK
- 정답 피드백에서 학습 기회 제공
- 첫 퀴즈 응답 시 SRS에 등록 (`introduced_by: quiz`)

---

## 7. SRS 연결

- 학습 탭에서 배운 단어 → SRS 등록 (`introduced_by: lesson`)
- 퀴즈 탭에서 만난 단어 → SRS 등록 (`introduced_by: quiz`)
- 두 경로 모두 같은 SRS 풀로 합류
- 복습 대상이 있는 유저: SRS 복습 위주 출제
- 복습 대상이 없는 유저: 새 단어 퀴즈 위주 출제

## 미결 사항 및 논의 필요 항목

### 1. 찍어서 맞춘 정답의 SRS 처리 (Codex 반박 — 치명)

**Codex 지적**: 4지선다는 25% 확률로 찍어 맞춤. 이걸 SRS에 "정답"으로 넣으면 복습 간격이 늘어나서 실제로는 모르는 단어가 안 나옴.

**선택지**:
- A) 현행: 정답은 정답. SRS가 알아서 교정 (단순)
- B) 반응시간 기반: 3초 이내 정답 = 확실히 앎, 8초 이상 = 찍었을 가능성 → SRS 반영 차등
- C) 임시 등록: 첫 정답은 `provisional`, 동세션 내 재출제(방향 반전 KR→JP)로 재검증 후 확정

**→ 재논의 필요. Codex는 C를 강력 추천.**

### 2. 문법 퀴즈 — 의미/뉘앙스 구분 모드 추가 여부 (Codex 반박)

**Codex 지적**: 재배열+조사 선택만으로는 "형태"만 훈련. 문법의 핵심인 "의미/뉘앙스 구분"이 빠짐.

**제안**: 3번째 모드 "문형 의미 구분" 추가 (재배열 45 / 조사 30 / 의미구분 25)

**→ 재논의 필요. 다만 모드가 3개로 늘면 복잡도 증가.**

### 3. 한자 단일 모드 리스크 (Codex 경고)

**Codex 지적**: 4지선다-읽기만으로는 인식 착시, 음독/훈독 맥락 구분 부족.

**제안**: 주 1회라도 문맥 속 읽기, 유사한자 구별을 섞어야 함.

**→ N4~ 확장 시 재검토. N5에서는 한자 비중 자체가 낮아 당장은 단일 모드로 시작 가능.**

### 4. 리스닝 — 음운 변별 훈련 부재 (Codex 지적)

**Codex 지적**: 지문/대화 기반만으로는 장단음, 촉음, 탁음 변별 훈련이 빠짐.

**제안**: 15~25%는 바텀업 드릴(짧은 단어/문장 변별) 추가.

예: おばさん vs おばあさん, きて vs きって 구분

**→ 재논의 필요. 지문 기반 + 변별 드릴 혼합이 이상적이나 복잡도 증가.**

### 5. 처음 보는 단어 출제 비율 상한 (미정)

한 세션에서 "처음 보는 단어"가 너무 많으면 찍기 게임이 됨.
신규 단어 비율 상한 (예: 세션의 30% 이내)을 정해야 함.

**→ SRS 엔진 설계 시 함께 결정**

### 6. "오늘의 퀴즈"와 리스닝의 관계 (미정)

리스닝은 이어폰 필요 → "오늘의 퀴즈"에 포함? 제외?
현재: 리스닝 탭에서 직접 진입만 가능. 단어/문법/한자 탭의 "오늘의 퀴즈"에는 리스닝 미포함.

**→ 확정 필요**
