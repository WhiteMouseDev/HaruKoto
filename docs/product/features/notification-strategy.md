# 알림 전략 설계서

> 최종 수정: 2026-03-13
> 상태: v1 구현 중

## 원칙

- 알림은 "알람 시계"가 아니라 "학습 유도 도구"
- 사용자 규모에 맞게 점진적 확장 (과잉 설계 금지)
- 주파수 제한으로 피로도 관리 우선

---

## v1: 로컬 알림 (현재 단계)

> 서버 의존 없이 기기 내에서 완결. 즉시 구현 가능.

### 1. 학습 리마인더
- **트리거**: 사용자가 설정한 시간 (기본값: 오전 9시)
- **구현**: `flutter_local_notifications` + iOS/Android 스케줄링
- **메시지 예시**: "오늘의 일본어, 준비됐어요!"
- **설정 UI**: 마이페이지 > 알림 설정 > 리마인더 시간 선택

### 2. 스트릭 방어 알림
- **트리거**: 자정 2시간 전 (22:00), 오늘 학습 기록이 없을 때
- **구현**: 매일 22:00 로컬 알림 스케줄, 학습 완료 시 당일 알림 취소
- **메시지 예시**: "연속 7일 학습 중! 오늘 놓치면 스트릭이 끊겨요 🔥"
- **조건**: 스트릭이 2일 이상인 경우에만 발송 (첫날부터 보내면 의미 없음)

### 3. 주파수 제한
- 하루 최대 2회 (리마인더 1회 + 스트릭 방어 1회)
- 리마인더 시간과 스트릭 방어 시간이 1시간 이내면 스트릭 방어만 발송

### 4. 설정
- 리마인더 ON/OFF (기본: ON)
- 리마인더 시간 선택 (기본: 09:00)
- 스트릭 방어 알림 ON/OFF (기본: ON)

## v2: FCM 푸시 알림 (사용자 100명+)

> 서버에서 발송. 앱 미실행 상태에서도 도달.

### 1. FCM 푸시
- **인프라**: Firebase Cloud Messaging (iOS APNs + Android FCM)
- **토큰 관리**: 로그인 시 FCM 토큰 서버 전송, 갱신 시 업데이트
- **발송**: FastAPI 서버에서 `firebase-admin` SDK로 발송

### 2. 이벤트 기반 알림
- **레벨업**: 경험치 누적으로 레벨 달성 시 즉시 푸시
- **업적 달성**: 첫 퀴즈 완료, 100문제 달성 등 마일스톤
- **스트릭 마일스톤**: 7일, 30일, 100일 연속 학습 축하
- **구현**: 각 도메인 로직에서 알림 생성 → notifications 테이블 저장 + FCM 발송

### 3. 동적 메시지
- **템플릿 시스템**: 알림 타입별 메시지 템플릿 + 사용자 데이터 바인딩
- **예시**: "어제 틀렸던 '{word}' 복습이 준비됐어요"
- **예시**: "레벨 {level}까지 경험치 {remaining}포인트 남았어요!"

### 4. 채널 분기 (간단 버전)
- 앱 포그라운드 → 인앱 토스트 (푸시 미발송)
- 앱 백그라운드/종료 → FCM 푸시

## v3: 개인화 (사용자 1만명+)

> 데이터 기반 최적화. 충분한 사용자 행동 데이터 축적 후.

### 1. 통계 기반 발송 시간
- 사용자별 최근 2주 평균 접속 시간 계산
- 평균 접속 시간 30분 전에 리마인더 발송
- 데이터 부족 시 기본값(설정 시간) 사용

### 2. 채널 라우팅 + 폴백
- 인앱 → 푸시 → 이메일 순서로 폴백
- 7일 이상 미접속 장기 휴면 유저 → 이메일 전환

### 3. 알림 그룹화
- 동일 유형 알림 병합 (예: "3명의 친구가 레벨업했어요")
- 알림 센터 내 날짜별/유형별 그룹 UI

### 4. A/B 테스트
- 메시지 문구, 발송 시간, 이모지 유무 등 변수 테스트
- 클릭률(CTR) 기반 자동 최적화

## 기술 스택 요약

| 단계 | 기술 | 비고 |
|------|------|------|
| v1 | `flutter_local_notifications` | 서버 불필요, 기기 내 완결 |
