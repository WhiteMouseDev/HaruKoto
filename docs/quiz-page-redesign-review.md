# 퀴즈 페이지 개편 설계 리뷰

> Date: 2026.03.17
> Reviewer: Claude Code (20년차 개발자 시점)
> 원본: Claude AI 작성 "하루코토 퀴즈 페이지 개편 설계 문서 v1.0"

## 총평: 7.5/10

방향성은 옳지만 실행 세부사항에 함정이 있다.

## 동의하는 방향

1. **학습(Input) / 퀴즈(Output) 분리** — 이미 우리가 하고 있는 구조의 명문화
2. **어순 모드를 문법으로 이동** — 어순은 문법 능력 검증이 맞음
3. **한자는 콘텐츠 준비 후** — 현실적 판단
4. **콘텐츠가 구조를 결정한다** — 빈 탭 방지 원칙 정확

## 주요 문제점

### 1. 세그먼트 컨트롤 시기
Phase 1에서 `[단어]` 하나뿐인 세그먼트는 UX 낭비. Phase 2에서 문법 스마트 퀴즈 준비되면 그때 도입.

### 2. 워딩 변경 리스크
- "자유 퀴즈 → 맞춤 연습": 동의
- "학습한 단어 → 학습 기록": 보류 ("학습한 항목"이 더 적절)
- "단어장 → 북마크": 반대 (한국 유저에게 "단어장"이 익숙, 단어장 유지)

### 3. 숙련도 기반 모드 자동 배분
현재 퀴즈 아키텍처는 세션 전체가 하나의 모드. 문제별 모드 전환은 전면 개편 필요.
별도 RFC로 분리 추천.

### 4. 문장배열 통합 마이그레이션
study_stages의 content_ids가 다른 테이블을 가리키는 문제. 단순 category enum 변경으로 해결 안 됨.

### 5. Phase 1 범위 과대
어순 모드 이동 + 문장배열 편입은 백엔드 로직 변경 수반. "즉시 적용"이 아님.

### 6. 빠진 것
- 타이핑 모드 처리 방향 미결정
- 가나 퀴즈와의 관계 미설명
- daily_goal JSONB 변경 (카테고리별 독립 목표) 스키마 누락

## 실행 계획 (리뷰어 제안)

### Phase 0 (0.5일): 워딩 + 메뉴 추가
- "자유 퀴즈" → "맞춤 연습"
- 문법 연습 바로가기 추가
- 행동/조회 영역 구분선

### Phase 1 (1~2일): 어순 모드 정리
- 단어 스마트 퀴즈에서 어순 제거
- 문법 퀴즈에 어순 전담
- study_stages는 변경하지 않음 (마이그레이션 회피)

### Phase 2 (2~3일): 문법 스마트 퀴즈
- `[단어 | 문법]` 세그먼트 도입
- SM-2 기반 문법 스마트 퀴즈
- daily_goal JSONB 전환
- 오답노트/학습기록 카테고리 필터

### Phase 3 (콘텐츠 의존): 한자
- kanjis 테이블 + user_kanji_progress
- 세그먼트 확장

### 보류: 숙련도 기반 모드 자동 배분
Phase 2 완료 후 사용자 데이터 보고 결정.
