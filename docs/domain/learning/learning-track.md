# 학습 트랙 (Learning Track) 설계

> 최종 수정: 2026-03-20
> 상태: 설계 확정 (일부 미결 사항 하단 참조)

---

## 1. 목적

**"처음 보는 일본어를 체계적으로 배우는 경로"**

- 대상: 유저 A "체계적 학습자" — 커리큘럼을 따라 차근차근 배우는 사람
- 목표: 매일 10~20분 투자 → 성취감 → JLPT 합격
- 학습 탭과 퀴즈 탭은 **독립적으로 작동** (퀴즈 탭만 쓰는 유저도 지원)

---

## 2. 마이크로 레슨 구조

### 1레슨 = 10분 완결

```
Phase 1: 도입 (3분)
  - 새 단어 5~8개 카드 (단어 + 읽기 + 뜻 + TTS + 이미지)
  - 문법 1개 카드 (패턴 + 의미 + 예문 2개)

Phase 2: 맥락 (3분)
  - 대화문 1개 (3~4턴, 위 단어+문법 포함)
  - TTS 재생 + 핵심 표현 하이라이트

Phase 3: 확인 (4분)
  - 5문항:
    1. 인식: 의미 선택 4지선다 (근접 오답 포함)
    2. 인식: 문장 의미/의도 선택
    3. 적용: 상황에 맞는 표현 선택
    4. 적용: 토큰 재배열 (문법 패턴)
    5. 산출유사: 토큰뱅크 문장 구성 (미끼 토큰 30~40% 포함)
  - 산출유사 문항: 선택지 노출 전 2~3초 회상 구간

Phase 4: 마무리 (30초)
  - 오늘 배운 것 요약 카드
  - "내일 복습 예약됨" 안내
```

### 단어 수 기준

| 상황 | 단어 수 |
|------|--------|
| 기본 | 5~7개 |
| 단순 어휘 (숫자, 요일, 색깔 등) | 8~10개 |
| 문법 난이도 높은 레슨 | 5~6개 (자동 감산) |
| 상한 | 10개 (복습형 레슨에서만) |

### 지문 형태

| 유형 | 설명 | 사용 시점 |
|------|------|----------|
| 상황 대화 (Type A) | 2인 대화, 3~4턴 | Part 1~3 전체 |
| 설명 대화 (Type B) | 1인 독백/나레이션 | Part 1 초반 (자기소개 등) |

## 3. 커리큘럼 구조

### JLPT 기준 + 주제 묶음

```
N5 (총 약 160 레슨)
│
├── Part 1: 기초 (레슨 1~30)
│   ├── Ch.1 인사와 소개 (5레슨)
│   ├── Ch.2 나와 가족 (5레슨)
│   ├── Ch.3 숫자와 시간 (5레슨)
│   ├── Ch.4 일상생활 (5레슨)
│   ├── Ch.5 장소와 위치 (5레슨)
│   └── Ch.6 음식과 쇼핑 (5레슨)
│
├── Part 2: 확장 (레슨 31~70)
│   ├── Ch.7 교통과 이동
│   ├── Ch.8 학교와 직장
│   ├── Ch.9 취미와 여가
│   ├── Ch.10 날씨와 계절
│   ├── Ch.11 건강과 병원
│   ├── Ch.12 요청과 허가
│   ├── Ch.13 경험과 계획
│   └── Ch.14 비교와 설명
│
├── Part 3: 실전 (레슨 71~100)
│   └── Part 1~2 어휘/문법을 조합한 실전 대화 지문
│
└── 보충 레슨 (레슨 101~160)
    ├── 빈출 어휘 보충
    └── 모의고사 형식 종합 문제
```

### JLPT 합격까지 예상 기간

```
하루 2레슨 (20분) × 주 5일 = 약 16주 (4개월)
하루 1레슨 (10분) × 주 5일 = 약 32주 (8개월)
```

## 4. 한자 처리

- **N5**: 별도 트랙, 레슨에서는 후리가나로 자연 노출
- **N4~**: 한자 비중 증가에 따라 레슨 내 자연 포함 검토
- 후리가나 점진적 제거 계획 필요 (TBD)

---

## 5. SRS 연결

- **레슨 완료 → SRS 등록**
- 레슨에서 틀린 문항의 단어/문법 → SRS에서 우선 복습
- 학습/퀴즈 양쪽에서 등록된 아이템은 같은 SRS 풀로 합류
- SRS 등록 시 `introduced_by: lesson` 태그

---

## 6. Unlock 조건

| 단위 | 조건 |
|------|------|
| 다음 레슨 | 이전 레슨 1회 완료 |
| 다음 챕터 | 이전 챕터 레슨 전체 완료 + 챕터 체크 통과 |
| 보강 루프 | 미달 시 보강 레슨/퀴즈 자동 추천 |

## 미결 사항 및 논의 필요 항목

### 1. SRS 등록 게이트 (Codex 반박)

**Codex 지적**: "레슨 완료 → 즉시 SRS 등록"은 잘못 학습한 항목까지 장기 복습으로 고정시킨다. 레슨 5문항 중 찍어 맞춘 것도 "배웠다"로 처리되는 문제.

**선택지**:
- A) 현행 유지: 즉시 등록, SRS가 알아서 자주 출제해서 교정 (PM 확정안)
- B) 게이트 추가: 레슨 완료 + 확인 문제 일정 점수 이상 시에만 등록
- C) 임시 등록: 모든 아이템 등록하되 `provisional` 상태, 다음 세션에서 재검증 후 확정

**→ 재논의 필요**

### 2. 아이템 상태 머신 정의 (미확정)

Codex 지적: `unseen / introduced / learning / mastered / lapsed` 상태 정의가 필요.
V3 문서에 있었으나 오늘 대화에서 재확정하지 않음.

**→ 03-SRS-ENGINE.md에서 설계 필요**

### 3. 후리가나 점진적 제거 (Scaffolding Fade)

Codex 지적: N5에서 항상 후리가나를 보여주면 한자 인지 전이가 늦어진다.
레벨/숙련도에 따라 후리가나를 줄여가는 계획이 필요.

**→ 한자 설계 시 함께 논의**

### 4. 레슨 통과 기준 상세

현재: "레슨 1회 완료"가 unlock 조건이지만, 통과 점수 컷(예: 5문항 중 3개 이상)이 미정.

**→ 재논의 필요**

### 5. 학습 이론 추가 반영

Codex 제안:
- **Testing Effect**: 인식형 과다 시 효과 급감, 회상 강도 확보 필요
- **Desirable Difficulty**: 너무 쉬운 문제 반복 금지
- **Feedback Quality**: 정오만이 아닌 "왜 틀렸는지" 제공

**→ 확인 문제 설계 고도화 시 반영**
