## 1. 현재 상태 및 문제점

### 현재 퀴즈 유형
| 유형 | 형태 | 대상 |
|------|------|------|
| 4지선다 (뜻 맞추기) | 일본어 → 한국어 뜻 선택 | 단어, 문법 |
| 가나 인식 | 가나 → 로마자 선택 (4지선다) | 히라가나, 가타카나 |
| 발음 매칭 | 로마자 → 가나 선택 (4지선다) | 히라가나, 가타카나 |
| 히라↔가타 매칭 | 히라가나 → 가타카나 선택 (4지선다) | 히라가나 ↔ 가타카나 |

### 핵심 문제
1. **모든 퀴즈가 4지선다** — 단조로움, 학습 깊이 부족
2. **가나 매칭에서 4:4 동시 출제 후 1:1** — UX 혼란
3. **생산형(Productive) 퀴즈 부재** — 인식만 하고 직접 만들어보는 연습이 없음
4. **문맥 없는 단어 학습** — 고립된 단어만 출제, 문장 속 활용 연습 부재
5. **SM-2 알고리즘 한계** — 최신 FSRS 대비 20-30% 비효율적

## 2. 추가할 퀴즈 유형

### 2-1. 매칭 페어 (순차 탭 매칭)

**방식**: Duolingo식 순차 탭. 좌측/우측 카드를 번갈아 탭하여 매칭. 정답 시 사라지고 남은 것 계속 진행.

**적용 대상**: 가나(문자↔발음), 단어(일본어↔뜻), 히라↔가타

**UI 구조**:
```
  일본어            뜻
┌────────┐    ┌────────┐
│  犬     │    │ 고양이  │
└────────┘    └────────┘
┌────────┐    ┌────────┐
│  猫     │    │  개    │  ← 선택됨(하이라이트)
└────────┘    └────────┘
┌────────┐    ┌────────┐
│  鳥     │    │  새    │
└────────┘    └────────┘

플로우: 犬 탭 → 개 탭 → ✓ 매칭! (페이드아웃) → 남은 카드 계속
```

**세부 사항**:
- 한 라운드당 4~6쌍
- 오른쪽 컬럼은 매번 랜덤 셔플
- 매칭 성공 시 초록 하이라이트 + 축소 사라짐 애니메이션
- 매칭 실패 시 빨간 쉐이크 + 선택 해제
- 모바일 한 손 조작 최적화 (한 컬럼 탭 → 다른 컬럼 탭)

**가나 매칭 개선** (기존 4:4 문제 해결):
- 기존: 4개 동시 출제 → 혼란
- 개선: 순차 탭 매칭으로 전환, 5~6쌍씩 직관적으로 진행

### 2-2. 빈칸 채우기 (Cloze)

**방식**: 문장 속 빈칸에 올바른 단어/조사를 보기에서 선택

**적용 대상**: 문법(조사, 접속사), 단어(문맥 속 활용)

**UI 구조**:
```
禁━たばこ ___ 吸ってはいけない。
(담배를 피워서는 안 된다)

이 문장의 빈칸에 들어갈 말은?

┌────┐  ┌────┐  ┌────┐
│ を  │  │ に  │  │ で  │
└────┘  └────┘  └────┘

정답: を ✓
해설: 禁じる는 대상(を)을 취합니다
```

**세부 사항**:
- 보기 2~4개 (난이도에 따라 조절)
- 정답 선택 후 해설 표시 (문법 포인트 설명)
- 한국어 번역 함께 제공 (학습 보조)
- 레벨별 난이도:
  - N5: 기본 조사 (は, が, を, に, で, へ)
  - N4: 접속표현 (~て, ~たり, ~ながら)
  - N3+: 복합 표현, 경어

**데이터 구조**:
```typescript
type ClozeQuestion = {
  sentence: string;        // "禁━たばこ {blank} 吸ってはいけない。"
  translation: string;     // "담배를 피워서는 안 된다"
  blankPosition: number;   // 빈칸 위치 (토큰 인덱스)
  correctAnswer: string;   // "を"
  options: string[];       // ["を", "に", "で"]
  explanation: string;     // "禁じる는 대상(を)을 취합니다"
  grammarPoint?: string;   // "조사 を의 용법"
};
```

### 2-3. 어순 배열 (문장 조립)

**방식**: 한국어 문장을 보고 일본어 단어 카드를 올바른 순서로 배열

**적용 대상**: 문법 (어순, 조사, 동사 활용)

**UI 구조**:
```
"나는 학생입니다"를 일본어로 만드세요:

정답 영역: [          ] [          ] [          ] [          ]

보기 카드:
┌──────┐ ┌──┐ ┌──────┐ ┌────┐
│ です  │ │は│ │がくせい│ │わたし│
└──────┘ └──┘ └──────┘ └────┘

탭 순서: わたし → は → がくせい → です
결과: わたしはがくせいです ✓
```

**레벨별 난이도 자동 조절**:
| 레벨 | 분리 방식 | 예시 |
|------|----------|------|
| N5 초급 | 의미 단위 묶음 | `[わたしは]` `[がくせいです]` |
| N5 중급 | 명사/조사 분리 | `[わたし]` `[は]` `[がくせい]` `[です]` |
| N4 | 동사/형용사 분리 | `[きのう]` `[は]` `[あめ]` `[が]` `[ふりました]` |
| N3+ | 활용어미까지 분리 | `[たべ]` `[ませ]` `[ん]` `[でした]` |

**데이터 구조** (토큰 배열):
```typescript
type SentenceToken = {
  text: string;           // "わたし"
  type: 'noun' | 'particle' | 'verb' | 'adjective' | 'copula' | 'adverb' | 'suffix';
  meaning: string;        // "나"
  order: number;          // 정답 순서
};

type SentenceArrangeQuestion = {
  koreanSentence: string;          // "나는 학생입니다"
  tokens: SentenceToken[];         // 토큰 배열
  difficultyLevel: 'beginner' | 'intermediate' | 'advanced';
  jlptLevel: JlptLevel;
  grammarPoint?: string;
};
```

**레벨별 토큰 병합 로직**:
- 프론트엔드에서 `difficultyLevel`에 따라 토큰을 동적으로 병합/분리
- 예: beginner → `noun + particle`을 하나의 카드로 병합

### 2-4. 글자 입력 (단계별 혼합)

**방식**: 난이도에 따라 글자 뱅크 → 인앱 키보드 → 로마자 입력으로 단계 상승

**적용 대상**: 가나 쓰기, 단어 스펠링, 읽기(리딩)

#### Stage 1: 글자 뱅크 선택 (초급)
```
"먹다"를 일본어로 쓰세요:

입력: [  ] [  ] [  ]

보기:
┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐
│べ│ │た│ │る│ │か│ │め│
└──┘ └──┘ └──┘ └──┘ └──┘

탭 순서: た → べ → る
결과: たべる ✓
```

- 정답 글자 + 오답 글자 2~3개 혼합
- 탭하면 입력 영역에 순서대로 채워짐
- 백스페이스로 마지막 입력 취소 가능

#### Stage 2: 인앱 가나 키보드 (중급)
```
"먹다"를 일본어로 쓰세요:

입력: た__

┌───────────────────────┐
│ あ  か  さ  た  な    │
│ い  き  し  ち  に    │
│ う  く  す  つ  ぬ    │
│ え  け  せ  て  ね    │
│ お  こ  そ  と  の    │
│      [⌫]  [확인]     │
└───────────────────────┘
```

- 앱 내장 커스텀 키보드 (OS 키보드 전환 불필요)
- 50음도 기반 배열
- 탁음(゛)/반탁음(゜) 토글 버튼
- 장음(ー)/소문자(ゃゅょっ) 지원

#### Stage 3: 로마자 입력 → 자동 변환 (고급)
```
"먹다"를 일본어로 쓰세요:

입력창: [taberu_____]

유저 타이핑: "ta" → た (자동변환)
유저 타이핑: "be" → べ (자동변환)
유저 타이핑: "ru" → る (자동변환)

결과: たべる ✓
```

- 영문 QWERTY 키보드 그대로 사용
- 헵번식 로마자 → 가나 실시간 변환
- 입력창 아래 힌트: `"ta"를 입력해보세요`
- 변환 규칙: `shi→し, chi→ち, tsu→つ` 등 표준 헵번식

**자동 단계 전환 조건**:
- 해당 단어의 SRS 마스터리가 Guru 이상 → 다음 Stage로 전환
- 사용자가 수동으로 난이도 선택도 가능

## 4. SRS 알고리즘 업그레이드: SM-2 → FSRS

### FSRS (Free Spaced Repetition Scheduler) 개요

SM-2 대비 20-30% 적은 복습으로 동일한 기억률 달성. Anki 23.10+의 기본 알고리즘.

### 핵심 파라미터

```typescript
type FSRSParams = {
  stability: number;      // 기억 안정성 (일 단위)
  difficulty: number;     // 항목 난이도 (0-10)
  retrievability: number; // 현재 기억 확률 (0-1)
  targetRetention: number; // 목표 기억률 (기본 0.9 = 90%)
};

type ReviewGrade = 1 | 2 | 3 | 4;
// 1: Again (다시) — 완전히 잊음
// 2: Hard (어려움) — 간신히 기억
// 3: Good (좋음) — 약간 망설임
// 4: Easy (쉬움) — 즉시 기억
```

### 마이그레이션 전략

1. **기존 SM-2 데이터 보존**: `easeFactor`, `interval` 유지
2. **FSRS 필드 추가**: `stability`, `difficulty`, `retrievability`
3. **초기값 변환**: SM-2 `interval` → FSRS `stability` 매핑
4. **점진적 전환**: 새 복습부터 FSRS 적용, 기존 데이터는 자연스럽게 전환

### DB 스키마 변경 (UserVocabProgress / UserGrammarProgress)

```prisma
model UserVocabProgress {
  // 기존 필드 유지
  easeFactor    Float     @default(2.5)
  interval      Int       @default(0)
  nextReviewAt  DateTime?

  // FSRS 추가 필드
  fsrsStability   Float?    // 기억 안정성
  fsrsDifficulty  Float?    // 항목 난이도 (0-10)
  lastGrade       Int?      // 마지막 평가 등급 (1-4)
}
```

### 마스터리 단계 시각화

| 단계 | 조건 | 색상 | 아이콘 |
|------|------|------|--------|
| 새 단어 | 미학습 | 회색 | ○ |
| 학습 중 | stability < 1일 | 빨강 | ● |
| 익숙함 | stability 1-7일 | 주황 | ●● |
| 숙달 | stability 7-30일 | 초록 | ●●● |
| 마스터 | stability 30일+ | 금색 | ★ |

model ClozeQuestion {

## 8. 기술 구현 참고사항

### 컴포넌트 구조
```
components/features/quiz/
├── multiple-choice.tsx      # 기존 4지선다 (리팩토링)
├── matching-pair.tsx        # 순차 탭 매칭
├── cloze-quiz.tsx           # 빈칸 채우기
├── sentence-arrange.tsx     # 어순 배열
├── character-bank-input.tsx # 글자 뱅크 입력
├── kana-keyboard.tsx        # 인앱 가나 키보드
├── romaji-input.tsx         # 로마자 → 가나 변환 입력
└── quiz-session.tsx         # 세션 래퍼 (유형별 분기)
```

### API 구조
```
api/v1/quiz/
├── start       # POST — 세션 시작 (quizType 파라미터로 유형 분기)
├── answer      # POST — 답변 제출 (inputMethod 필드 추가)
├── complete    # POST — 세션 완료
├── recommend   # GET  — SRS 기반 오늘의 추천 (NEW)
└── stats       # GET  — 통계

api/v1/questions/
├── cloze       # GET — 빈칸 채우기 문제 조회
├── arrange     # GET — 어순 배열 문제 조회
└── matching    # GET — 매칭 페어 문제 조회
```

- `hooks/use-quiz.ts` — 쿼리/뮤테이션 훅 확장
- `lib/query-keys.ts` — 새 퀴즈 타입 키 추가
- `lib/show-events.ts` — 게임 이벤트 표시
- `lib/api.ts` — `apiFetch()` 유틸
- `components/ui/` — 기존 shadcn 컴포넌트 활용
