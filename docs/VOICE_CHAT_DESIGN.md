# AI 음성 회화 기능 설계서

> 작성일: 2026-03-02
> 상태: 설계 검토 중

## 1. 개요

현재 텍스트 기반 AI 회화에 **음성 입출력**을 추가하여, 실제 말하기 연습이 가능한 학습 환경을 구현한다.

### 핵심 원칙

- **정확성 우선**: 교육앱이므로 STT 오인식으로 인한 학습 혼란 방지
- **피드백 루프 유지**: 사용자가 말한 내용에 대한 교정/피드백이 핵심
- **점진적 고도화**: 턴제 음성 → 발음 평가 → 실시간 대화 순서로 확장

---

## 2. 대화 방식: 턴제 음성 대화

### 왜 실시간이 아닌가?

| 항목 | 실시간 음성 | 턴제 음성 (채택) |
|------|-----------|----------------|
| STT 정확도 | 비원어민 발음 오인식 빈번 | 변환 후 확인/수정 가능 |
| 문법 피드백 | 대화 흐름 끊김 → 제공 불가 | 턴마다 피드백 제공 |
| 학습 효과 | 초급자 압박감 → 학습 저해 | 생각할 시간 확보 |
| 비용 | 높음 ($0.06~0.24/분) | 낮음 (STT+TTS 건당 과금) |
| 구현 복잡도 | WebRTC/WebSocket 실시간 처리 | REST API + 파일 업로드 |
| 기존 코드 활용 | 전면 재설계 필요 | ChatInput 확장으로 구현 |

### 사용자 플로우

```
┌─────────────────────────────────────────────┐
│  기존 텍스트 채팅 흐름 (유지)                    │
│                                             │
│  ┌─────┐    ┌──────┐    ┌─────┐    ┌─────┐ │
│  │ 입력 │ →  │ 전송  │ →  │ AI  │ →  │표시  │ │
│  │(키보드)│    │      │    │응답  │    │     │ │
│  └─────┘    └──────┘    └─────┘    └─────┘ │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  음성 채팅 흐름 (신규)                          │
│                                             │
│  ┌─────┐    ┌──────┐    ┌──────┐    ┌─────┐│
│  │🎤탭  │ →  │ 녹음  │ →  │ STT  │ →  │텍스트││
│  │      │    │      │    │ 변환  │    │표시  ││
│  └─────┘    └──────┘    └──────┘    └──┬──┘│
│                                        │    │
│                         ┌──────────────┘    │
│                         ▼                   │
│  ┌─────┐    ┌──────┐    ┌──────┐           │
│  │ 🔊  │ ←  │ TTS  │ ←  │ 확인/ │           │
│  │재생  │    │ 응답  │    │ 수정  │→ 전송     │
│  └─────┘    └──────┘    └──────┘           │
└─────────────────────────────────────────────┘
```

**단계별 상세:**

1. **녹음**: 마이크 버튼 탭 → 녹음 시작 (파형 시각화)
2. **STT 변환**: 녹음 종료 → 서버로 오디오 전송 → 일본어 텍스트 변환
3. **텍스트 확인**: 변환된 텍스트를 입력창에 표시 → 사용자가 수정 가능
4. **전송**: 기존 텍스트 전송과 동일 흐름 (기존 `/api/v1/chat/message` 활용)
5. **AI 응답**: 텍스트 + TTS 음성 동시 제공
6. **TTS 재생**: 스피커 아이콘 탭 시 AI 응답 음성 재생

## 3. 기술 아키텍처

### 3-1. 전체 구조

```
┌────────────────────────────────┐
│          Client (Web)          │
│                                │
│  MediaRecorder API → Blob      │
│  Audio Element ← TTS URL       │
│                                │
│  ChatInput (기존) + VoiceInput  │
└────────────┬───────────────────┘
             │ POST /api/v1/chat/voice
             │ (multipart: audio file)
             ▼
┌────────────────────────────────┐
│          Server (API)          │
│                                │
│  1. Audio → STT (Whisper)      │
│  2. Text → LLM (기존 흐름)      │
│  3. AI Response → TTS          │
│  4. Return: text + audio URL   │
└────────────────────────────────┘
```

### 3-2. STT (Speech-to-Text)

**추천: OpenAI Whisper API**

| 항목 | Whisper API | Google Cloud STT | Web Speech API |
|------|-----------|-----------------|---------------|
| 일본어 정확도 | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| 비원어민 대응 | 우수 | 보통 | 미흡 |
| 비용 | $0.006/분 | $0.016/분 | 무료 (Chrome only) |
| 브라우저 호환 | 서버 처리 (100%) | 서버 처리 (100%) | Chrome/Edge만 |
| 오프라인 | 불가 | 불가 | 불가 |

**선택 이유:**
- Whisper는 비원어민 일본어 인식에서 가장 높은 정확도
- 서버 사이드 처리 → 브라우저 호환성 문제 없음
- 비용이 가장 저렴 (10분 대화 = ~$0.06)
- Vercel AI SDK와 동일 OpenAI 생태계

**구현:**
```typescript
// POST /api/v1/chat/voice
const transcription = await openai.audio.transcriptions.create({
  file: audioFile,
  model: 'whisper-1',
  language: 'ja',           // 일본어 지정 → 정확도 향상
  prompt: scenarioContext,   // 시나리오 컨텍스트 → 도메인 정확도 향상
});
```

### 3-3. TTS (Text-to-Speech)

**추천: OpenAI TTS API**

| 항목 | OpenAI TTS | Google Cloud TTS | Web Speech API |
|------|----------|-----------------|---------------|
| 일본어 자연스러움 | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| 음성 종류 | 6종 (alloy, echo 등) | 다수 | OS 의존 |
| 비용 | $0.015/1K자 | $0.016/1K자 | 무료 |
| 속도 | 스트리밍 지원 | 스트리밍 지원 | 즉시 |
| 브라우저 호환 | 서버 처리 (100%) | 서버 처리 (100%) | 대부분 지원 |

**선택 이유:**
- 자연스러운 일본어 발화 (학습자 리스닝 모델로 적합)
- Whisper와 동일 SDK → 코드 단순화
- 스트리밍 지원 → 응답 대기 시간 단축

**구현:**
```typescript
// 선택적 재생: 사용자가 스피커 아이콘 탭 시에만 생성
const speech = await openai.audio.speech.create({
  model: 'tts-1',
  voice: 'nova',      // 여성 음성 (일본어에 적합)
  input: messageJa,
  speed: 0.9,         // 학습자를 위해 약간 느리게
});
```

### 3-4. 비용 추정

| 항목 | 단가 | 1회 대화 (5분, 10턴) | 월 1,000회 |
|------|------|---------------------|-----------|
| STT (Whisper) | $0.006/분 | ~$0.03 | ~$30 |
| TTS (선택 재생 50%) | $0.015/1K자 | ~$0.02 | ~$20 |
| LLM (기존) | - | 기존 비용 | 기존 비용 |
| **음성 추가 비용** | | **~$0.05/회** | **~$50/월** |

## 4. 신규 API 설계

### 4-1. POST `/api/v1/chat/voice` — 음성 메시지 전송

**Request** (multipart/form-data):
```
audio: File (webm/wav, max 10MB)
conversationId: string
```

**Response:**
```json
{
  "transcription": "こんにちは、東京タワーに行きたいです",
  "aiResponse": {
    "messageJa": "東京タワーですね！...",
    "messageKo": "도쿄타워군요! ...",
    "feedback": [...],
    "hint": "...",
    "newVocabulary": [...]
  },
  "audioUrl": null
}
```

**처리 흐름:**
1. 오디오 파일 수신
2. Whisper API → 일본어 텍스트 변환
3. 기존 `/chat/message` 로직 재활용 (텍스트 기반 AI 응답)
4. 응답 반환 (TTS는 별도 엔드포인트)

### 4-2. POST `/api/v1/chat/tts` — AI 응답 음성 생성

**Request:**
```json
{
  "text": "東京タワーですね！素晴らしい選択です。",
  "speed": 0.9
}
```

**Response:** `audio/mpeg` 바이너리 스트림

**설계 이유:**
- TTS를 별도 엔드포인트로 분리 → 선택적 재생 지원
- 사용자가 스피커 아이콘을 탭할 때만 TTS 호출 → 비용 절감
- 캐싱 가능 (같은 텍스트 재요청 시)

### 4-3. POST `/api/v1/chat/voice/transcribe` — STT만 수행 (편집 모드용)

**Request** (multipart/form-data):
```
audio: File
scenarioContext?: string
```

**Response:**
```json
{
  "transcription": "こんにちは"
}
```

**용도:** 사용자가 녹음 후 텍스트 확인/수정만 하고, 전송은 기존 텍스트 전송으로 하는 경우.

## 5. 클라이언트 구현

### 5-1. 신규 컴포넌트

#### `VoiceInput` — 음성 녹음 컴포넌트

```
┌──────────────────────────────────────┐
│  ┌────┐                    ┌─────┐  │
│  │ 🎤 │  ████████░░░ 0:03  │ ✕   │  │
│  │    │                    │취소  │  │
│  └────┘                    └─────┘  │
│         녹음 중... 말씀해주세요         │
└──────────────────────────────────────┘

녹음 완료 후:
┌──────────────────────────────────────┐
│  "こんにちは、東京に行きたいです"        │
│                                      │
│  ┌──────┐  ┌──────┐  ┌──────────┐   │
│  │ 🔄   │  │ ✏️   │  │  전송 →  │   │
│  │다시녹음│  │ 수정  │  │          │   │
│  └──────┘  └──────┘  └──────────┘   │
└──────────────────────────────────────┘
```

**상태 머신:**
```
idle → recording → transcribing → preview → idle
                                    ↓
                                  editing → idle
```

- `idle`: 마이크 버튼 표시
- `recording`: 파형 시각화 + 타이머 + 취소 버튼
- `transcribing`: 로딩 스피너 ("변환 중...")
- `preview`: 변환된 텍스트 표시 + 다시녹음/수정/전송 버튼
- `editing`: 텍스트 직접 편집 모드

#### `AudioPlayer` — TTS 재생 컴포넌트

```
AI 메시지 말풍선 내부:
┌────────────────────────────────┐
│  東京タワーですね！              │
│  素晴らしい選択です。            │
│                                │
│  🔊 재생  ▶ ━━━━━━━░░░ 0:03   │
│  🐌 느리게 재생                 │
└────────────────────────────────┘
```

**기능:**
- 일반 속도 재생 (0.9x)
- 느리게 재생 (0.7x) — 학습자용
- 재생 위치 표시 (프로그레스 바)
- 재생 중 텍스트 하이라이트 (선택적, 추후)

### 5-2. 기존 컴포넌트 수정

#### `ChatInput` 확장

```
기존:
┌──────────────────────────────┐
│  일본어로 입력하세요...    🔡  │
└──────────────────────────────┘

변경 후:
┌──────────────────────────────┐
│  일본어로 입력하세요...  🎤 🔡 │
└──────────────────────────────┘
        마이크 버튼 추가 ↑
```

- 키보드 입력과 음성 입력을 토글
- 마이크 탭 → VoiceInput 모드로 전환
- 키보드 탭 → 기존 텍스트 입력 모드로 복귀

#### `ChatMessage` 확장

- AI 메시지에 `AudioPlayer` 추가
- 스피커 아이콘: 탭 시 TTS API 호출 → 재생
- 첫 재생 시 오디오 생성 → 이후 캐시 사용

## 6. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 오디오 녹음 | MediaRecorder API | webm/opus 포맷 |
| 오디오 재생 | HTML5 Audio API | mp3 스트리밍 |
| 파형 시각화 | Web Audio API (AnalyserNode) | 실시간 파형 |
| STT | OpenAI Whisper API | 서버 사이드 |
| TTS | OpenAI TTS API | 서버 사이드, 선택적 호출 |
| LLM | 기존 유지 (Gemini/GPT) | 변경 없음 |
| 파일 업로드 | FormData + fetch | multipart/form-data |
| 마이크 권한 | navigator.mediaDevices | getUserMedia |

## 7. 마이크 권한 & 에러 처리

### 권한 플로우

```
마이크 버튼 탭
    ↓
navigator.mediaDevices.getUserMedia({ audio: true })
    ↓
┌─ 허용 → 녹음 시작
├─ 거부 → "마이크 사용을 허용해주세요" 안내 + 설정 링크
└─ 미지원 → "이 브라우저에서는 음성 입력을 지원하지 않습니다" 안내
```

### 에러 시나리오

| 시나리오 | 처리 |
|---------|------|
| 마이크 권한 거부 | 안내 메시지 + 텍스트 입력 유지 |
| STT 변환 실패 | "음성을 인식하지 못했어요. 다시 시도해주세요" + 재녹음 |
| STT 결과가 일본어가 아님 | "일본어로 말해주세요" 안내 |
| 녹음 10초 초과 | 자동 종료 + 변환 |
| TTS 생성 실패 | 스피커 버튼 비활성 + 텍스트만 표시 |
| 네트워크 오류 | 재시도 버튼 표시 |

## 10. 현재 코드와의 통합 포인트

### 변경 없음 (기존 코드 재활용)

| 파일 | 이유 |
|------|------|
| `/api/v1/chat/message` | 음성이든 텍스트든 최종 입력은 텍스트 → 기존 API 그대로 사용 |
| `/api/v1/chat/start` | 시나리오 시작 흐름 변경 없음 |
| `/api/v1/chat/end` | 대화 종료 + 피드백 흐름 변경 없음 |
| `packages/ai/src/prompts.ts` | 시스템 프롬프트 변경 없음 |
| `FeedbackScores`, `FeedbackDetails` | 피드백 컴포넌트 변경 없음 |

### 수정 필요

| 파일 | 변경 내용 |
