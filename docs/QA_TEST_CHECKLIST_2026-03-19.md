# QA 테스트 체크리스트 (2026-03-19)

이번 세션에서 수정된 항목 + 기존 기능 회귀 테스트 목록입니다.

---

## 1. 인증 (Authentication)

### Apple Sign-In
- [ ] Apple로 계속하기 버튼 탭 → iOS 시스템 바텀시트 표시
- [ ] Apple ID로 로그인 → 온보딩 페이지 이동 (신규 유저)
- [ ] 온보딩 완료 후 홈 진입
- [ ] 앱 재실행 → 스플래시 → 홈 (자동 로그인)

### Google / Kakao Sign-In
- [ ] Google로 계속하기 → 로그인 성공 → 홈
- [ ] Kakao로 계속하기 → 로그인 성공 → 홈

### 이메일 로그인
- [ ] "이메일로 계속하기" 버튼 → 이메일 폼 펼쳐짐
- [ ] 회원가입 모드 → 비밀번호 확인 필드 표시
- [ ] 비밀번호 불일치 → "비밀번호가 일치하지 않습니다" 에러
- [ ] 잘못된 자격 → 한국어 에러 메시지 (JSON 노출 안 됨)

### 온보딩
- [ ] 신규 가입 → 온보딩 페이지로 이동 (홈 직행 안 됨)
- [ ] 온보딩 완료 후 → 홈 진입

---

## 2. 로그인/스플래시 UI

### 스플래시
- [ ] 앱 실행 → 벚꽃잎 나타남 (화면 곳곳에서 fade-in)
- [ ] 꽃잎이 살랑살랑 흔들림 (위에서 떨어지지 않음)
- [ ] 로고 + 태그라인 애니메이션 정상
- [ ] 3초 후 로그인 또는 홈으로 이동

### 로그인 페이지
- [ ] 벚꽃잎 배경 무한 루프 (끊김 없이 계속)
- [ ] 로고/태그라인 스타일이 스플래시와 동일
- [ ] OAuth 버튼 순서: Apple(검정) → Google(흰) → Kakao(노랑) → 구분선 → 이메일(아웃라인)

---

## 3. 홈 화면

### 미션 카드
- [ ] 오늘의 미션 3개 표시 (label, description 포함)
- [ ] 미션 진행률 반영
- [ ] 미션 카드 위치: Study Card 아래

### Study Card (QuickStartCard)
- [ ] 단어 탭 → "오늘의 단어" 탭 → 바텀시트 (학습 분배 미리보기)
- [ ] 바텀시트에서 "학습 시작" → 퀴즈 진입
- [ ] 문법/문장 탭 → `/practice`로 이동 (기존 동작)

---

## 4. 단어 퀴즈

### 4지선다
- [ ] 퀴즈 시작 → 문제 표시
- [ ] 답 선택 → 정답/오답 피드백
- [ ] 마지막 문제 완료 → **"퀴즈 결과 저장에 실패했어요" 에러 안 뜸**
- [ ] 결과 화면 → 정확도, XP, 레벨 표시

### 매칭/빈칸/문장배열
- [ ] 각 모드 진입 → 문제 표시 → 완료 → 결과 화면

### 퀴즈 결과
- [ ] 오답 노트 탭 → 오답 목록 표시
- [ ] 단어장 저장 동작

---

## 5. 가나 학습

### 스테이지 학습
- [ ] 스테이지 선택 → 학습 카드 표시
- [ ] 퀴즈 시작 → 문제 풀기
- [ ] 퀴즈 완료 → XP 부여 + 정확도 표시 (**실패 안 뜸**)
- [ ] 스테이지 완료 → 다음 스테이지 해금

### TTS
- [ ] 가나 문자 옆 스피커 버튼 → 발음 재생
- [ ] 첫 재생 후 2회째 → 즉시 재생 (캐시)

---

## 6. TTS (Text-to-Speech)

### 단어 TTS
- [ ] 단어 스피커 버튼 → 발음 재생
- [ ] 발음 속도 자연스러움 (speed 1.0)
- [ ] 단어 억양이 이전보다 안정적 (stability 0.85)
- [ ] 첫 재생 → GCS 캐시 → 2회째 즉시

### ElevenLabs 동작
- [ ] TTS 재생 시 Sentry에 에러 없음
- [ ] ElevenLabs 실패 시 Gemini 폴백 동작

---

## 7. 결제

### 결제 내역
- [ ] MY → 결제 내역 → **로그아웃 안 됨**
- [ ] 결제 내역 목록 정상 표시 (날짜, 금액, 상태)
- [ ] 결제 내역 없으면 빈 상태 표시

### 푸터 연락처
- [ ] MY 페이지 하단 → 연락처 `010-8595-9869` 표시
- [ ] 이메일 `whitemousedev@whitemouse.dev` 표시

## 10. 바텀시트 회귀 테스트

### 닉네임 변경
- [ ] MY → 닉네임 편집 → 키보드 열린 상태에서 저장
- [ ] 레드스크린 / `_dependents.isEmpty` assertion 없음
- [ ] 시트 닫힘 → 닉네임 반영

### 채팅 통화 설정
- [ ] 실전회화 → 설정 아이콘 → 값 변경 → 저장
- [ ] 레드스크린 없음, 시트 1회만 닫힘
- [ ] 재진입 시 저장된 값 유지

### 하루 목표 변경
- [ ] 홈 → QuickStartCard 목표 편집 → 다른 값 선택
- [ ] 스낵바 1회만 표시, 시트 정상 닫힘
- [ ] 목표 값 반영
