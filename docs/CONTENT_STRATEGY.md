## 3. 퀴즈 문제 생성 전략

### 3.1 문제 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **뜻 맞추기** | 일본어 → 한국어 뜻 | 食べる → ? (먹다/마시다/자다/걷다) |
| **읽기 맞추기** | 한자 → 히라가나 읽기 | 食べる → ? (たべる/のべる/とべる) |
| **한국어→일본어** | 한국어 → 일본어 | 먹다 → ? (食べる/飲む/寝る) |
| **빈칸 채우기** | 문장 속 빈칸 | 朝ごはんを＿＿ます。(食べ) |
| **문법 적용** | 문법 패턴 적용 | 〜てください: 食べる → ? (食べてください) |

### 3.2 문제 생성 방법

```
1. 정적 문제 (미리 생성)
   - N5 단어 800개 × 문제유형 5종 = 약 4,000문제
   - 직접 제작 + AI 보조 생성 후 수동 검수
   - DB에 저장, 랜덤 출제

2. 오답 선택지 생성 규칙
   - 같은 품사의 단어에서 선택
   - 같은 JLPT 레벨에서 선택
   - 비슷한 의미/발음의 단어 우선 (혼동 유도)
   - 정답 위치는 매번 랜덤

3. 난이도 조절
   - 정답률 기반 적응형 출제
   - 자주 틀리는 문제 가중치 높여서 복습
   - 에빙하우스 망각 곡선 기반 복습 주기
```

## 4. 시드 데이터 제작 계획

### 4.1 Sprint 1에서 준비할 최소 데이터

| 항목 | 수량 | 제작 방식 |
|------|------|----------|
| N5 단어 | 200개 (전체의 25%) | 공개 데이터 + 수동 검수 |
| N5 문법 | 30개 (전체의 37%) | 직접 제작 |
| 단어 퀴즈 | 800문제 (200×4유형) | AI 생성 + 수동 검수 |
| 문법 퀴즈 | 120문제 (30×4유형) | AI 생성 + 수동 검수 |
| 회화 시나리오 | 5개 | 직접 기획 |

### 4.2 데이터 제작 파이프라인

```bash
# 1. 공개 데이터에서 N5 단어 추출
scripts/extract-jmdict-n5.ts

# 2. 한국어 뜻 추가 (AI 보조)
scripts/add-korean-meanings.ts

# 3. 예문 생성 (AI 보조)
scripts/generate-examples.ts

# 4. 퀴즈 문제 생성 (AI 보조)
scripts/generate-quizzes.ts

# 5. 수동 검수 후 DB 시딩
scripts/seed-database.ts
```

## 6. 콘텐츠 업데이트 계획

### 6.1 정기 업데이트
| 주기 | 내용 |
|------|------|
| 주 1회 | 새 퀴즈 문제 추가 (AI 생성 + 검수) |
| 월 1회 | 새 회화 시나리오 추가 (2~3개) |
| 분기 1회 | 새 JLPT 레벨 데이터 추가 |

### 6.2 사용자 피드백 반영
```
- 오답률 높은 문제 → 힌트/설명 보강
- 사용자 신고 문제 → 수동 검수 후 수정
- AI 회화 피드백 분석 → 시나리오 개선
```
