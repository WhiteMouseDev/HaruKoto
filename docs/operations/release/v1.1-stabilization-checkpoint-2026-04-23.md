## Manual UAT Checklist

Manual UAT should focus on real product behavior rather than another architecture pass.

- [ ] Auth and session: app launch, session restore, login provider flows available in the target environment, `ensure-user`, logout, re-login.
- [ ] Onboarding and profile: new-user onboarding, profile load, daily goal update, kana visibility update, JLPT level update, call settings update, app restart persistence.
- [ ] Voice call success path: start call, complete with transcript, feedback generation, feedback screen navigation, no stuck analyzing state.
- [ ] Voice call no-data path: short or silent call shows the no-transcript/no-data state instead of a blank failure.
- [ ] Voice call generation-failure path: backend feedback failure shows a retry/error state instead of silent completion.
- [ ] Study core flow: lesson start, quiz answer, completion, result screen, wrong-answer handling, wordbook save.
- [ ] Mobile API regression: 401 refresh/retry behavior, network error message, response-shape compatibility for touched endpoints.
- [ ] Admin smoke: content list/detail, review queue, per-field TTS play/regenerate, backend `{ detail }` surfaced in the localized error toast.
- [ ] Observability: no uncaught Sentry/error-log spikes for the tested flows.
