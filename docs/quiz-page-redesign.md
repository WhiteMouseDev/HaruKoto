# 퀴즈 페이지 개편 검토

## 1. 현재 구조 (개편 후)

```
┌─────────────────────────────────┐
│  퀴즈                            │
│                                  │
│  ┌───────────────────────────┐  │
│  │  오늘의 학습 / 이어서 학습   │  │
│  │  하루 목표 20개 · 7/20     │  │
│  │                      ▶    │  │
│  └───────────────────────────┘  │
│                                  │
│  오답노트                     →  │
│  학습한 단어                   →  │
│  단어장                       →  │
│  ─────────────────────────────  │
│  자유 퀴즈                    →  │
│                                  │
└─────────────────────────────────┘
```

- 원 버튼 CTA → 바텀시트에서 학습 정보 + 시작
- 미완료 세션 있으면 CTA가 "이어서 학습하기"로 변경
- 자유 퀴즈는 메뉴 → 바텀시트 (레벨/유형/모드 선택)
- 탭 없음

---

## 2. 말해보카 스타일

```
┌─────────────────────────────────┐
│  [어휘] [문법] [표현] [실전회화] [리스닝] │  ← 상단 카테고리 탭
│                                  │
│  ┌───────────────────────────┐  │
│  │  어휘 학습                  │  │
│  │  하루 목표 20개  ✏️    ▶    │  │
│  └───────────────────────────┘  │
│                                  │
│  내가 좋아하는 문장            →  │
│  내가 좋아하는 오답노트        →  │
│  내가 학습한 어휘              →  │
│  내 단어장                    →  │
│  AI 질문 답변 모아보기         →  │
│                                  │
└─────────────────────────────────┘
```

- 상단 카테고리 탭 5개 — 탭마다 독립 CTA + 메뉴
- 각 카테고리가 충분한 콘텐츠 볼륨을 가짐
- CTA 누르면 바텀시트에서 세부 정보(배분, 진도) 표시

---

## 3. 우리가 현재 지원하는 학습 카테고리

| 카테고리 | 콘텐츠 현황 | 스마트 퀴즈 | 퀴즈 모드 | 비고 |
|---------|-----------|-----------|---------|------|
| **단어 (VOCABULARY)** | N5 15스테이지, 7500+ 단어 | O | 4지선다, 매칭, 빈칸, 어순 | 핵심 카테고리 |
| **문법 (GRAMMAR)** | N5 10스테이지 | X (자유 퀴즈만) | 4지선다, 빈칸, 어순 | 매칭 미지원 |
| **문장배열 (SENTENCE)** | N5 6스테이지 | X | 어순 전용 | 학습 탭 스테이지 방식 |
| **가나 (KANA)** | 히라가나/가타카나 | X | 별도 플로우 | 초보 전용, 퀴즈 탭 X |
| **한자 (KANJI)** | DB enum만 존재 | X | 미구현 | 콘텐츠 없음 |
| **리스닝 (LISTENING)** | DB enum만 존재 | X | 미구현 | 콘텐츠 없음 |

### 기타 학습 기능 (퀴즈 외)
- **AI 실전회화**: 여행/일상/비즈니스/자유 시나리오 (별도 탭으로 이미 분리됨)
- **학습 탭**: 스테이지 기반 구조화 학습 (단어/문법/문장배열/가나)

---

## 4. 탭 도입 시 비교 분석

### 지금 탭을 넣는다면?

```
[단어]  [문법]
```

**문제점:**
- 2개 탭은 탭 UI의 가치가 낮음 (토글 버튼이면 충분)
- 문법은 스마트 퀴즈 미지원 → 탭 전환해도 CTA 동작이 달라야 함
- 문장배열은 탭에 넣기엔 콘텐츠 볼륨 부족 (6스테이지)
- 말해보카의 5탭과 비교하면 빈약해 보임

### 탭이 의미 있는 시점

다음 조건 중 2개 이상 충족 시:
- [ ] 카테고리 3개 이상 (예: 단어 + 문법 + 한자)
- [ ] 각 카테고리가 스마트 퀴즈 지원
- [ ] 카테고리별 콘텐츠가 N5~N1까지 충분

### 로드맵 예시

```
Phase 1 (현재):  단어 중심 원 버튼 구조
Phase 2:         문법 스마트 퀴즈 지원 → [단어] [문법] 토글 or 탭
Phase 3:         한자/리스닝 콘텐츠 추가 → [단어] [문법] [한자] [리스닝] 탭 전환
```

## 5. 현재 단계 추천 구조

### Option A: 현행 유지 (원 버튼 + 메뉴)
- 장점: 단순, 인지 부하 최소
- 단점: 문법이 자유 퀴즈 메뉴 안에 묻힘
- **적합 시점**: 단어 중심 MVP, 문법 사용률 낮을 때

### Option B: CTA 카드에 카테고리 토글 추가
```
┌───────────────────────────────┐
│  [단어]  [문법]                │  ← 카드 내부 토글
│                               │
│  오늘의 학습                   │
│  하루 목표 20개 · 7/20    ▶   │
└───────────────────────────────┘
```
- 장점: 문법도 1등 시민으로 승격, 구조 변경 최소
- 단점: 문법 스마트 퀴즈 미지원이라 CTA 동작 분기 필요
- **적합 시점**: 문법 스마트 퀴즈 구현 직후

### Option C: 말해보카식 상단 탭
- 장점: 확장성 최고, 카테고리별 독립 UX
- 단점: 현재 2개 카테고리로는 과한 구조
- **적합 시점**: 카테고리 3개 이상 + 각각 충분한 콘텐츠

---

## 6. 결론

**현재**: Option A 유지가 적합. 단어 학습이 핵심이고 문법은 보조.

**다음 마일스톤**: 문법 스마트 퀴즈 구현 시 Option B로 전환.

**장기**: 한자/리스닝 콘텐츠 확보 시 Option C (말해보카식 탭).

핵심 원칙: **콘텐츠가 구조를 결정한다.** 콘텐츠 없이 구조만 먼저 만들면 빈 탭이 생기고 앱이 빈약해 보임.
